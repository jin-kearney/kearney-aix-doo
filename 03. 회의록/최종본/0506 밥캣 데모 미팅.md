**논의사항**
- **사전질의서 관련 피드백**
	- 현재 사전질의서는 전사 단위 질문으로 구성되어 있어 Function 및 데이터 Type별로 상이한 답변 도출됨
	- Function(재무, 생산, 품질, R&D) 및 데이터 Type별로 데이터 수집, 관리, 활용 방식이 상이하여 데이터 현황을 일반화하여 답변하기 어려움  
	- → Use-case 또는 Scope 기준으로 질의 시 구체적 답변 가능
	- 현재 사전질의서 내용이 데이터 관리 관점과 AI 활용 관점이 혼재되어 있음  
		- **데이터 관리:** 저장 위치, 카탈로그, 접근 권한
		- **AI** **활용:** 파싱 가능 여부(PDF, Excel-병합셀, 색상), 데이터 구조, Chunking 가능 여부
	- 밥캣에서 이해하고 있는 AI Ready 데이터는 LLM/GenAI가 별도 커스터마이징 없이 이해 가능한 데이터 구조
	- 데이터 카탈로그, 접근성 등은 데이터 관리 영역이며 AI Ready 데이터와 구분 필요
- **밥캣 AI/데이터 관련 내부 진행 상황**
	- 현재 Structured Data 및 Unstructured Data를 AI 활용을 위해 가공하는 작업은 내부적으로 Proactive하게 진행 중
	- 기존 데이터 그대로 활용할 경우 AI 성능 한계가 존재하여 데이터 처리 로직을 별도로 구현하는 방식으로 대응하고 있음
	- GraphDB는 내부 검토 및 일부 적용 중이며, Ontology는 플랫폼 안정성 이슈로 현재 Hold 상태
-  **AI Ready Data** **체계에 대한 밥캣의 의견**
	- 현재 데이터를 그대로 활용할 경우 AI 성능 한계가 존재하며, 이를 보완하기 위해 Custom 솔루션을 적용하는 경우가 있음  
	- → 다만 Custom 솔루션이 아닌 기존 데이터 구조 변경을 통해 해결 가능한 영역도 존재함
- 지주 차원에서 그룹 공통 가이드라인과 Function별 Standard를 체계화하여 배포 및 Communication하는 방향 필요
