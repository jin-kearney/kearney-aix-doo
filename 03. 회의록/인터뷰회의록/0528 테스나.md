□	두산 테스나 AI/Data 과제
	AVI Verify 자동화 시스템 구축
	과제 설명: AVI는 Auto Visual Inspection의 약자로, 자동 외관 검사 장비.
해당 장비가 외관을 일차적으로 스캔해서 Pass와 Fail 구분한 것을 이차적으로 사람이 직접 진성 Fail인지 가성 Fail인지 Verify해야 함. 이를 알고리즘을 통해 자동으로 판정할 수 있는 시스템을 구축하고자 함.
	진행 상황: 작년 PoC 진행 후 효과성 검증 실시 및 사장님 보고 완료. 하지만 비즈니스 환경상 장비가 추가로 들어와야 하는 것들이 있어서 투자 집행은 안 돼 있고 현재까지 홀딩 돼 있는 상태. 
	추진 조직: 품질팀에서 추진하고 디지털혁신팀이 서포트하고 있는 형식.
	PoC 데이터 취합 과정: AVI 장비에 Rule-base로 Fail이라고 판단되는 것은 AVI 장비에 남고 이는 Verify하는 작업자 PC에 연결되어 있음. PoC는 작업자 PC에 있는 Verify 이미지, 진성 Fail과 가성 Fail 기록을 취합해 진행함.
	PoC 데이터 분량: 약 10,000장 (진성 Fail 5%, 가성 Fail 95%)
	PoC 결과: 진성 Fail과 가성 Fail은 잘 구분하지만, 불량 유형은 구분하지 못함. 불량 유형은 사람이 다시 레이블링 필요함.
	Trouble Shooting Agent
	과제 설명: 장비 운영 시 고장이 발생하면 장비 관리자가 조치를 취해야 하는데 신입 오퍼레이터가 조치를 취해야 할 경우 이를 빠르게 처리하기 위해 과거 조치 이력을 찾아주거나 매뉴얼을 검색해서 이미지를 보여주는 Agent를 RAG 형태로 구축하고자 함. 해당 과제는 FAQ 10개 정도로, 추론보다는 흩어져 있는 자료를 모아서 요약 후 원하는 답변이 나오도록 하는 것이 목적임.
	PoC 데이터 취합 과정: Trouble Shooting Agent를 위해 뭐가 필요한지 몰라서 일단 모든 데이터를 모으려고 했음. 현업을 통해 Microsoft Teams 공유 폴더, 개인 폴더 등에 존재하는 데이터 및 매뉴얼 위치를 찾아 한곳에 모으고 이를 하나의 엑셀 파일에 취합함. 매뉴얼의 파일 양식이 PDF, Word, 동영상. Excel 등 다양했으며, 매뉴얼의 이미지는 캡처 후 엑셀에 붙여 넣는 방식의 수작업으로 진행함. 데이터 취합 과정은 약 2주가 걸림. 
	진행 상황: 작년에 추진하려고 했던 것으로, PoC 이후 진행하고 있지 않음.
	PoC 결과: 솔루션 2개로 검증 진행, Agent의 응답 속도가 느리고 아키텍처가 이상함. 응답 속도가 느린 이유로는 Semantic Layer 등의 미흡한 구성, 현재 형태의 온톨로지나 데이터 정리가 안 되어 있던 점, 안정화되어 있지 않은 솔루션으로 추정함.

	(커니) 트러블 슈팅 에이전트의 응답 속도가 왜 느렸던 것인지? 
	(이재천 수석) 솔루션 2개로 검증을 해봤는데 지금 형태의 온톨로지나 데이터 정리가 안 되어 있었기 때문에 느렸다고 생각함. 솔루션도 안정화되어 있는 상태도 아니었음.
	(커니) RAG를 위한 학습 데이터를 모을 때 Pain Point는 없었는지?
	(성정화 수석) 데이터를 모으는 것이 힘들었음. 현업에 물어봐서 데이터 및 매뉴얼 위치를 찾고 한곳에 모아서 PoC를 진행함. 팀별로 데이터 위치가 팀즈 공유 폴더인 경우도 있었고 개인 폴더인 경우도 있었음.
	(커니) 매뉴얼의 파일 양식이 달랐을 것 같은데 전처리를 하지는 않았는지?
	(이재천 수석) 안 했음. 매뉴얼의 파일 양식이 PDF, Word, 동영상. Excel 등 다양했는데 트러블 슈팅을 할 때 뭐가 필요한지 몰라서 일단 모았음. 모은 파일들은 하나의 엑셀 파일에다가 모으는 작업을 했음.
	(커니) 하나의 엑셀로 모으는 것은 어떻게 진행한 것인지?
	(이재천 수석) 매뉴얼에 있는 이미지는 캡처해서 엑셀에 붙여 넣는 방식으로 하나하나 수작업으로 복사/붙여 넣기를 함. 당시 과제는 FAQ로, 질문을 10개 했을 때 원하는 답변이 나오는 방식을 원했기 때문에 답변이 될만한 데이터를 모은 것임. 추론이라는 개념보다 흩어져 있는 자료를 모아서 요약 후 잘 답변하게 하는 목적임. 해당 과정은 2주 정도 걸림.
	(커니) AVI Verify 자동화 PoC과정에서 데이터를 어떻게 모았는지?
	(이재천 수석) AVI 장비에 Rule-base로 Fail이라고 판단되는 것은 AVI 장비에 남고 Verify 하는 작업자 PC에 연결되어 있음. 그래서 작업자 PC에 있는 Verify 이미지를 가져와서 PoC를 진행함. 작업자 PC에 진성 Fail과 가성 Fail 기록이 남아 있어서 학습시킬 수 있음.
	(커니) 어느 정도 분량을 PoC진행한 것인지?
	(이재천 수석) 기간은 모르겠으나 분량은 10,000장 정도임. 
	(커니) 진성Fail과 가성Fail의 데이터 양이 차이가 나는지?
	(이재천 수석) 진성 Fail이 5% 정도, 나머지 95%는 가성 Fail임. PoC 결과 진성 Fail과 가성 Fail은 잘 구분하는데 불량 유형은 구분하지 못했음. 불량 유형은 사람이 다시 레이블링해야 학습에 사용할 수 있을 것 같음.
	(커니) 현재 데이터를 모으거나 AI과제를 할 때 오너십이나 드라이브는 디지털혁신팀이 가져가는지?
	(이재천 수석) 맞음.
	(커니) AI과제를 진행할 때 디지털혁신팀이 현업팀에 업무를 지정해서 요청하는지?
	(성정화 수석) 맞음. 각 현업에 특정 업무를 시킬 사람을 지정해달라고 요청함. 
	(커니) 내부적으로 데이터 관련 팀과 AI 관련 팀이 디지털혁신팀으로 합쳐져 있는 것인지?
	(성정화 수석) 맞음. DDI는 필요한 데이터를 뽑아주는 역할만 함.
	(커니) 그룹 내에서 AX팀 같은 타팀과의 관계는 어떻게 되는지?
	(성정화 수석) AX 센터와 DE 센터에 계약이 돼 있어서 지원받는 걸로 되어 있는데 아직 서로 소통이 많이 된 건 아님.
	(성정화 수석) 커니에서 진행 중인 프로젝트의 산출물이 어떻게 되는지?
	(커니) 각 20개 주제별 가이드와 데이터 관리 조직 같은 거버넌스 잡는 것을 목표로 하고 있음.
	(두산지주) 고객사와의 관계 때문에 시스템이 거의 온프레미스로 두고 있어 자체적으로 활용하기 어려운 것으로 알고 있는데 맞는지?
	(성정화 수석) 제품군이 여러 가지가 있는데 온프레미스여야 하는 것은 스마트 제품군이라고 카드에 들어가는 IC칩밖에 없음. 나머지는 클라우드에 올려도 괜찮은 걸로 확인해서 PoC하고 있는 것을 클라우드로 전환하려고 진행 중임. 고객과의 계약 때문은 아니고 보안 인증 협회에서 온프레미스로 해야 한다고 정해서 클라우드에 못 올리는 것임.
	(커니) 새로운 데이터의 클라우드 업로드 가능 여부 같은 것의 확인 절차는 어떻게 되는지?
	(성정화 수석) 디지털혁신팀에 보안 담당자가 있고 프로젝트 진행 시에는 설계한 아키텍처가 보안 정책에 위배되지 않는지 지주사 SA 팀에 확인받음.
