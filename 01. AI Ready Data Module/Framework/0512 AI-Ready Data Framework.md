### Pillar A. Findable

#### A-1. 데이터 카탈로그

**주제 설명:** AI와 사용자가 데이터 자산의 존재, 위치, 오너, 접근 경로를 찾을 수 있도록 등록하는 자산 목록 체계

**MECE 점검:** "어디에 무엇이 있는가"만 다룸. 의미는 A-2에서

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
|1|어떤 데이터 자산을 카탈로그에 등록해야 하는가?|데이터 자산 등록 대상 기준, 우선 등록 데이터 목록|
|2|데이터 자산의 위치와 접근 경로를 어떻게 기록할 것인가?|카탈로그 필수 등록 항목|
|3|여러 시스템에 흩어진 데이터를 하나의 카탈로그로 어떻게 통합할 것인가?|시스템별 카탈로그 연계 방식|
|4|AI나 사용자가 필요한 데이터를 어떻게 쉽게 찾게 할 것인가?|검색 분류 체계, 데이터 탐색 화면 구성 기준|
|5|카탈로그를 최신 상태로 어떻게 유지할 것인가?|카탈로그 갱신 프로세스, 미등록 데이터 점검 기준|

---

#### A-2. 메타데이터

**주제 설명:** 데이터 자산의 구조, 형식, 단위, 생성 기준, 갱신 주기 등 기술·운영 속성을 설명하는 관리 정보

**MECE 점검:** 자산 단위(테이블/필드)의 속성만. 개념 정의는 A-3

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
|1|AI가 데이터 구조를 이해하려면 어떤 메타데이터가 필요한가?|AI-ready 메타데이터 표준|
|2|테이블명과 필드명만으로 의미를 알 수 없는 데이터를 어떻게 설명할 것인가?|필드 설명서, 컬럼 Dictionary|
|3|데이터 단위, 기준일, 집계 기준을 어떻게 명확히 할 것인가?|단위·기준일·집계 방식 정의서|
|4|메타데이터를 어떻게 자동 수집·갱신할 것인가?|메타데이터 자동 수집·승인 프로세스|

---

#### A-3. 비즈니스 Glossary

**주제 설명:** 조직 내 업무 용어, 약어, 동의어를 표준 정의로 통일해 데이터 해석의 일관성을 확보하는 용어 사전

**MECE 점검:** 단어 단위 정의만. 개념 간 관계는 B-3

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
|1|어떤 용어를 표준화 대상으로 삼을 것인가?|표준화 대상 용어 목록|
|2|표준 용어는 어떤 항목으로 정의해야 하는가?|Glossary 표준 템플릿|
|3|계열사나 부서 간 용어 충돌을 어떻게 조정할 것인가?|용어 충돌 조정 프로세스|
|4|Glossary를 AI 활용 체계와 어떻게 연결할 것인가?|Glossary-AI 검색 연계 방식|
|5|용어 변경을 어떻게 관리할 것인가?|용어 변경 관리 기준|

### Pillar B. Understandable

#### B-1. 데이터 전처리

**주제 설명:** 문서, 표, 이미지 등 비정형·반정형 데이터를 AI가 읽을 수 있는 구조화 형태로 변환하는 작업

**MECE 점검:** 형식 변환만. 품질 검증은 C-2, OCR 등 대량전환(디지털화)는 F-3

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
|1|어떤 데이터가 전처리 대상인가?|전처리 대상 데이터 유형 목록|
|2|문서 유형별로 무엇을 추출해야 하는가?|문서 유형별 추출 기준|
|3|추출된 내용을 어떤 구조로 변환할 것인가?|구조화 데이터 포맷, Chunking 기준|
|4|원천 문서 양식이 바뀌어도 전처리 결과를 어떻게 안정적으로 유지할 것인가?|전처리 테스트셋, 파서 변경 관리 기준|
|5|전처리 결과를 AI 활용 환경에 어떻게 제공할 것인가?|AI 입력용 전처리 산출물|

#### B-2. 데이터 해설/주석

**주제 설명:** AI 학습을 위해 데이터에 라벨, 분류, 해석 기준 등 사람이 부여한 의미 정보를 붙이는 체계

**MECE 점검:** 학습 데이터 라벨만. 평가용 정답은 E-3

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
|1|어떤 데이터에 라벨이나 주석이 필요한가?|주석 대상 데이터 목록|
|2|라벨 체계는 어떻게 설계할 것인가?|라벨 Taxonomy, 라벨 정의서|
|3|사람마다 다르게 해석할 수 있는 데이터를 어떻게 일관되게 주석 처리할 것인가?|Annotation Guideline, 사례집|
|4|AI가 생성한 주석을 사람은 어디까지 검토할 것인가?|AI-assisted Labeling 운영 기준|
|5|주석 데이터를 어떻게 버전 관리할 것인가?|라벨 버전 관리 기준, 학습 데이터셋 이력|

|6|온톨로지 변경을 어떻게 운영할 것인가?|온톨로지 변경 관리 기준|

### Pillar C. Trustworthy

#### C-1. Observability

**주제 설명:** 데이터 흐름의 지연, 누락, 변경, 이상 징후를 운영 중 감지하고 알림하는 모니터링 체계

**MECE 점검:** 실시간 모니터링만. 사후 추적은 C-3

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
|1|어떤 데이터 이상을 관찰해야 하는가?|관찰 지표 목록|
|2|이상 여부를 어떤 기준으로 판단할 것인가?|이상 탐지 룰, 임계값 기준|
|3|이상이 발생했을 때 누구에게 어떻게 알릴 것인가?|알림 기준, Severity Level|
|4|AI 결과 이상이 데이터 문제인지 모델 문제인지 어떻게 구분할 것인가?|데이터-모델-Prompt 관찰 체계|
|5|Observability 운영 수준을 어떻게 측정할 것인가?|Observability KPI|

---

|3|데이터 접근 권한을 어떻게 통제할 것인가?|접근 권한 정책, RBAC/ABAC 기준|

|6|품질/권한 관리를 어떻게 측정할 것인가?|품질/권한 KPI|

|3|AI 답변이나 보고서의 근거 데이터를 어떻게 역추적할 것인가?|AI 답변 근거 로그, Citation 구조|
|4|데이터 변경의 영향 범위를 어떻게 분석할 것인가?|영향도 분석 기준|
|5|감사나 규제 대응 시 무엇을 증빙할 것인가?|감사 대응 Lineage Report|

### Pillar D. Actionable

|#|Key Question|Lv. 2 세부내용|
|---|---|---|
