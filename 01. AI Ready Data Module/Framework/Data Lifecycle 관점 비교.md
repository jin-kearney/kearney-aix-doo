정리하면 이렇게 보는 게 가장 깔끔합니다.

> **기존 14개 + 추가 16개 = 총 30개**

> **기존 15개 + 추가 16개 = 총 31개**

# 4. 추천 기준

문서나 RFP에서 쓰려면 저는 **31개 기준**을 추천합니다.

이유는 `데이터 품질관리`와 `권한관리`는 실행 과제가 완전히 다릅니다.

|항목|실행 성격|
|---|---|
|데이터 품질관리|품질 룰, 검증, 정합성, 결측, 이상치, 모니터링|
|권한관리|접근통제, 보안, 개인정보, 역할 기반 권한, 감사|

따라서 최종 표현은 이렇게 가면 좋습니다.

> 기존 5 Framework의 14개 주제를 실행 관점에서 15개로 세분화하고,  
> Data Lifecycle 관점에서 16개 주제를 추가하여  
> 총 31개 실행 주제로 확장한다.
