# AI 데이터 거버넌스 프레임워크: 주제별 백서 구성 및 Key Question 답변 가이드

본 문서는 아래 두 파일만을 근거로, 각 주제별로 **백서에 담아야 할 핵심 내용**과 **Key Question에 대해 제시 가능한 답변 방향**을 정리한 실행 가이드입니다.

- 기준 문서 1: `AI 데이터 거버넌스 프레임워크 — 주제별 Key Questions.md`
- 기준 문서 2: `주제별 Level 정의.md`

---

## 1. Findable

### 1-1. 데이터 카탈로그
- 백서에 담을 내용
  - 분산 데이터 소스 인벤토리 및 통합 카탈로그 아키텍처
  - AI Agent 검색/연결을 위한 API·시맨틱 인덱스 설계
  - 스키마 변경 감지 포함 자동 등록·동기화 운영 체계
  - 카탈로그 KPI 및 운영 거버넌스
- Key Question 답변 방향
  - “통합·등록”은 자동 스캔 + 표준 메타모델 기반 중앙 카탈로그로 해결
  - “AI 자동 탐색·연결”은 시맨틱 검색/메타데이터 인덱싱으로 구현
  - “최신성 유지”는 변경 이벤트 기반 동기화와 드리프트 감지로 보장

### 1-2. 메타데이터
- 백서에 담을 내용
  - Technical/Business/Operational 3계층 메타데이터 모델
  - LLM 이해형 메타데이터 스키마(설명, 예시, 제약조건)
  - 프로파일링·NLP 기반 자동 추출/태깅
  - 표준(DCAT 등) 연계와 변경 감지·자동 갱신
- Key Question 답변 방향
  - “의미 설명”은 표준 템플릿 기반 기술+업무 메타데이터 결합으로 해결
  - “AI 맥락 이해”는 AI Readable Schema로 컨텍스트 제공
  - “자동 갱신”은 메타데이터 드리프트 탐지 및 워크플로우 자동화로 대응

### 1-3. 비즈니스 Glossary
- 백서에 담을 내용
  - 용어 수집·중복/충돌 진단 및 표준화 프로세스
  - 표준 용어 템플릿(정의·동의어·약어·맥락·책임자)
  - Glossary-데이터 자산 자동 연결 및 AI 주입 패턴
  - 제안→검토→승인→폐기 라이프사이클
- Key Question 답변 방향
  - “부서별 용어 통일”은 거버넌스 위원회 + 표준 템플릿으로 정렬
  - “AI 활용 연결”은 RAG/프롬프트 컨텍스트/임베딩 사전으로 구현
  - “변화 대응”은 버전·승인 기반 변경관리 프로세스로 일관성 확보

## 2. Understandable

### 2-1. 데이터 전처리
- 백서에 담을 내용
  - 비정형 데이터(PDF/표/이미지) 구조화 전략
  - 클렌징 규칙 라이브러리(결측·이상치·중복·정규화)
  - 표준 입력 스키마/Feature Store
  - 멱등성 전처리 파이프라인 + 품질 회귀 테스트/SLA
- Key Question 답변 방향
  - “비정형 변환”은 유형별 파서와 표준 포맷 변환 체계로 해결
  - “품질 처리 일관성”은 룰 라이브러리와 자동화 파이프라인으로 보장
  - “원천 변경 대응”은 회귀 테스트 + SLA 모니터링으로 안정화

### 2-2. 데이터 해설/주석
- 백서에 담을 내용
  - 라벨링 가이드라인 및 주석 스키마
  - LLM Pre-labeling + Human Review 하이브리드
  - IAA 기반 품질 측정(Cohen's Kappa 등)
  - Active Learning·Golden Dataset·버전관리
- Key Question 답변 방향
  - “대량 주석 효율화”는 자동 사전주석 + HITL로 처리량 확보
  - “일관성 확보”는 가이드라인 + IAA 지표 관리로 해결
  - “AI 주석 검증”은 샘플링 검수 범위·우선순위 정책으로 운영

### 2-3. 온톨로지
- 백서에 담을 내용
  - 도메인 개념/관계/속성 모델링 방법론
  - Knowledge Graph 구현 및 저장 기술 선택
  - LLM-KG 결합 패턴(GraphRAG 등)
  - 버전·확장·폐기 거버넌스
- Key Question 답변 방향
  - “지식·인과 구조화”는 온톨로지+KG로 형식화
  - “개념 관계 이해”는 Graph 기반 검색/추론으로 강화
  - “지속 업데이트”는 진화 관리 체계(버전/승인/확장)로 대응

## 3. Trustworthy

### 3-1. Observability
- 백서에 담을 내용
  - Freshness/Volume/Schema/Distribution/Lineage 지표 체계
  - 자동 이상탐지(통계·ML·LLM judge)
  - Data Drift vs Model Drift 분리 진단
  - RCA 자동화 + SLO/SLI 기반 대응 플레이북
- Key Question 답변 방향
  - “문제 단계 탐지”는 단계별 관측 지표 및 경보 설계로 해결
  - “무인 이상감지”는 통계+ML 탐지 파이프라인으로 구현
  - “데이터 vs 모델 구분”은 분리 진단 프레임워크로 수행

### 3-2. 데이터 품질/권한 관리
- 백서에 담을 내용
  - 품질 차원 정의와 측정 규칙
  - 학습 데이터 적격성 체크리스트(저작권·동의·민감도)
  - RBAC/ABAC 다층 접근통제 + Policy-as-Code
  - 접근 로그 감사 및 이상 접근 탐지
- Key Question 답변 방향
  - “부적합 데이터 차단”은 적격성 게이트로 사전 통제
  - “세밀 권한 통제”는 역할/속성 기반 정책 자동 집행으로 구현
  - “품질 자동관리”는 품질 규칙 엔진 + 대시보드로 운영

### 3-3. 데이터 계통(Lineage)
- 백서에 담을 내용
  - Table/Column/Feature 단위 라인리지 정의
  - OpenLineage 기반 자동 수집
  - Source→ETL→ML→AI Output E2E 추적
  - 영향도 분석 및 규제 대응 리포팅
- Key Question 답변 방향
  - “원인 역추적”은 backward tracing으로 신속 식별
  - “변환 이력 자동기록”은 표준 이벤트 수집으로 자동화
  - “감사 증빙”은 실시간 lineage 리포트로 즉시 대응

## 4. Governed

### 4-1. 데이터 운영관리
- 백서에 담을 내용
  - DataOps 성숙도 진단
  - 오케스트레이션 아키텍처
  - 장애 복구(재시도·백필·격리) 설계
  - Data Contract 기반 협업 + CI/CD for Data
- Key Question 답변 방향
  - “E2E 자동화”는 오케스트레이션 + 표준 운영모델로 달성
  - “장애 복구/영향 최소화”는 failure-aware 설계로 대응
  - “팀 협업 충돌 방지”는 계약 기반 인터페이스와 배포 표준으로 해결

### 4-2. 데이터 생명주기 관리
- 백서에 담을 내용
  - 분류체계·Retention 정책 매트릭스
  - 법적 보존/삭제 의무 매핑
  - Tiered Storage 자동 이관
  - 자동 폐기 증빙 및 비용 최적화
- Key Question 답변 방향
  - “폐기·이관 식별”은 분류/보존 정책 엔진으로 자동화
  - “법적 보존 구분”은 규제 매핑 룰 기반 관리로 해결
  - “비용 통제”는 저장 계층화 + 수명주기 정책으로 최적화

### 4-3. 데이터 디지털화
- 백서에 담을 내용
  - 디지털화 대상 인벤토리/우선순위
  - OCR/VLM 추출 파이프라인
  - 정확도 검증 체계(Sample QA, Double-entry)
  - Digital-First 업무 재설계
- Key Question 답변 방향
  - “수기·종이 전환”은 문서 AI + 표준 스키마 적재로 구현
  - “오류·누락 품질보장”은 이중검증 및 표본감사로 관리
  - “초기 디지털 생성”은 현장 프로세스 재설계로 제도화

### 4-4. 데이터 보안(민감정보 마스킹)
- 백서에 담을 내용
  - 비정형 민감정보 탐지(NER/Regex/LLM)
  - 마스킹 기법 카탈로그
  - Utility-Privacy 균형 설계
  - 목적/역할 기반 동적 마스킹 + 재식별 위험 평가
- Key Question 답변 방향
  - “민감정보 자동탐지”는 다중 탐지기 결합으로 정확도 확보
  - “성능 저하 균형”은 형식보존·가명처리 등 목적별 기법 선택으로 대응
  - “차등 적용”은 ABAC 기반 정책으로 수준별 제어

## 5. Evolvable

### 5-1. 데이터 Product화
- 백서에 담을 내용
  - Data Product 정의/분류와 Product Canvas
  - 셀프서비스 Discovery/Provisioning 플랫폼
  - Data Contract 표준화
  - Adoption/Reuse/NPS KPI
- Key Question 답변 방향
  - “셀프서비스 체계”는 데이터 제품 카탈로그+자동 프로비저닝으로 구현
  - “품질·갱신·책임자 명확화”는 Product Canvas/SLA로 제도화
  - “재사용 극대화”는 계약 기반 공유 표준과 KPI 관리로 달성

### 5-2. 합성데이터
- 백서에 담을 내용
  - 합성데이터 활용 시나리오
  - 도메인별 생성기법 선택 가이드
  - Fidelity/Privacy 평가 프레임워크
  - 편향·과적합 통제 거버넌스
- Key Question 답변 방향
  - “데이터 부족/제한 대응”은 목적별 합성데이터 생성 전략으로 해결
  - “패턴 반영 검증”은 통계유사성 + ML 효용평가로 검증
  - “편향/과적합 방지”는 위험지표 모니터링과 승인체계로 관리

### 5-3. AI 평가 데이터
- 백서에 담을 내용
  - 평가 차원 정의(정확성·신뢰성·안전성 등)
  - Golden Dataset 구축 방법
  - SME 평가 수집 워크플로우
  - LLM-as-a-Judge + Human Calibration + 지속평가
- Key Question 답변 방향
  - “평가 기준/정답 정의”는 업무 KPI와 연결된 다차원 기준으로 정립
  - “전문가 판단 축적”은 구조화된 평가 UX/프로세스로 데이터화
  - “지속 갱신”은 모델/업무 변경 이벤트 기반 버전업으로 운영

### 5-4. 데이터 파이프라인 Feedback Loop
- 백서에 담을 내용
  - Agent Telemetry 스키마
  - 피드백 수집 채널 설계
  - Feedback→Training 변환 파이프라인
  - 개선 주기·A/B 실험·점진 롤아웃
- Key Question 답변 방향
  - “결과/오류 자동수집”은 실행 트레이스·툴콜·결과 로그 표준화로 해결
  - “재학습/프롬프트 반영”은 변환 파이프라인과 트리거 정책으로 구현
  - “주기/자동화 설계”는 업무 중요도 기반 다주기 운영으로 최적화

## 6. Actionable

### 6-1. Physical 데이터
- 백서에 담을 내용
  - 현장 데이터 소스 인벤토리
  - OPC-UA/MQTT/Kafka 통합 수집
  - Edge 전처리·노이즈 필터링
  - Stream vs Batch 기준 및 Digital Twin 연계
- Key Question 답변 방향
  - “실시간 활용 수집·연결”은 표준 프로토콜 + 스트리밍 인프라로 구현
  - “노이즈·누락 정제”는 엣지 필터링 + 시계열 품질 룰로 해결
  - “실시간/배치 구분”은 지연 허용치·업무 임계도 기준으로 분기

### 6-2. API/Tool 연계 데이터
- 백서에 담을 내용
  - Tool 인벤토리/분류
  - Tool Schema 표준화
  - MCP 기반 Registry 아키텍처
  - Call Telemetry 자산화 + 보안/인증
- Key Question 답변 방향
  - “입출력 표준화”는 OpenAPI/JSON Schema 기반 공통 스펙으로 해결
  - “공통 프로토콜 관리”는 MCP Registry + 버전/폐기 정책으로 운영
  - “호출 이력 자산화”는 호출로그 표준 저장·검색 체계로 축적

### 6-3. Prompt/Harness 자산화
- 백서에 담을 내용
  - 업무 SOP→Prompt 구조화 방법
  - 작성 표준/템플릿 라이브러리
  - Prompt 버전관리 및 성능 회귀 테스트
  - Harness 표준(도구 연결·메모리·실행환경)
- Key Question 답변 방향
  - “업무 판단 구조화”는 Task Decomposition 기반 프롬프트 설계로 구현
  - “재사용 자산화”는 Git 기반 버전/카탈로그 운영으로 제도화
  - “성능 저하 대응”은 회귀테스트·모니터링·개선 루프로 해결
