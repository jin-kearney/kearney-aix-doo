따라서 3개 Era는 아래처럼 바꾸는 것이 좋습니다.

|단계|Era 명칭|의미|
|---|---|---|
|**1단계**|**Analysis-Ready Era**|사람이 데이터를 분석할 수 있도록 준비하는 시대|
|**2단계**|**Model-Ready Era**|AI 모델이 데이터를 이해·검색·생성·평가할 수 있도록 준비하는 시대|
|**3단계**|**Agent-Ready Era**|AI Agent가 데이터를 기반으로 판단·호출·실행·개선할 수 있도록 준비하는 시대|

# 1. 공통 기반 주제

## 모든 Era에서 공통적으로 필요한 데이터 주제

아래 주제들은 특정 시대에만 필요한 것이 아니라,  
**Analysis-Ready, Model-Ready, Agent-Ready 전 단계에 공통적으로 필요한 기반 요소**입니다.

다만 시대가 발전할수록 요구 수준이 높아집니다.

|공통 주제|공통적으로 필요한 이유|Era가 발전하면서 달라지는 점|
|---|---|---|
|**데이터 카탈로그**|데이터가 어디에 있고, 누가 관리하며, 어떻게 활용되는지 찾아야 함|분석가 검색 중심 → AI 검색 기반 → Agent의 자동 데이터 탐색 기반|
|**메타데이터**|데이터 구조, 속성, 생성 이력, 품질 기준을 설명해야 함|사람이 읽는 설명 → 모델이 이해하는 설명 → Agent가 실행 조건까지 파악하는 설명|
|**비즈니스 Glossary**|조직 내 업무 용어와 지표 정의를 일관되게 맞춰야 함|분석 용어 표준화 → AI 질의 해석 기준 → Agent 판단 기준|
|**데이터 품질 권한관리**|신뢰 가능한 데이터와 적절한 접근 권한이 필요함|분석 신뢰성 확보 → AI 답변 신뢰성 확보 → Agent 실행 권한 통제|
|**데이터 계통 Lineage**|데이터 출처, 변환, 이동 경로를 추적해야 함|분석 결과 추적 → AI 답변 근거 추적 → Agent 판단·실행 근거 추적|
|**Observability**|데이터와 파이프라인 상태, 이상 징후를 모니터링해야 함|데이터 오류 감지 → AI 활용 품질 감시 → Agent 실행 실패·이상 행동 감시|
|**데이터 운영관리**|데이터 공급, 배포, 활용 과정을 안정적으로 운영해야 함|분석 데이터 운영 → AI 데이터 운영 → Agent 실행 데이터 운영|
|**데이터 생명주기 관리**|데이터 생성부터 보관, 활용, 폐기까지 정책적으로 관리해야 함|저장·폐기 관리 → AI 학습/활용 데이터 관리 → Agent 사용 데이터의 지속 관리|
|**데이터 보안 개인·민감정보 마스킹**|민감정보 보호와 컴플라이언스를 확보해야 함|분석 접근 통제 → AI 입력/출력 보안 → Agent 실행 과정의 민감정보 통제|
|**데이터 Product화**|데이터를 반복 활용 가능한 자산으로 관리해야 함|분석용 데이터셋 → AI 활용 데이터셋 → Agent가 호출 가능한 데이터 서비스|

# 2. Era별 특화 주제

## 1단계. Analysis-Ready Era

### 과거: Data 분석 시대

이 단계의 핵심은 데이터를 **사람이 분석할 수 있는 형태로 확보하고 정리하는 것**입니다.  
즉, 데이터의 존재 여부, 디지털화, 정제, 기본 활용성이 중요합니다.

|특화 주제|이 Era에서 중요한 이유|
|---|---|
|**데이터 디지털화**|문서, 이미지, 오프라인 정보를 분석 가능한 디지털 데이터로 전환하는 것이 출발점|
|**Physical 데이터**|운영 시스템, 설비, 현장 등 실제 원천 데이터를 확보해야 분석이 가능|
|**데이터 전처리**|분석 목적에 맞게 데이터를 정제, 변환, 표준화해야 활용 가능|

### 한 줄 정의

> **Analysis-Ready Era는 데이터를 찾고, 디지털화하고, 정제하여 사람이 분석 가능한 상태로 만드는 단계**입니다.

---

## 2단계. Model-Ready Era

### 현재: AI 활용 시대

이 단계는 기존의 “AI-Ready”라는 표현 대신 **Model-Ready**로 부르는 것이 좋습니다.  
AI-Ready가 전체 프레임의 제목이라면, 2단계는 AI 모델이 데이터를 실제로 이해하고 활용할 수 있게 만드는 단계이기 때문입니다.

이 단계의 핵심은 데이터를 **AI 모델이 이해·검색·생성·평가할 수 있는 형태로 만드는 것**입니다.

|특화 주제|이 Era에서 중요한 이유|
|---|---|
|**데이터 해설/주석**|데이터의 의미, 맥락, 정답 기준을 태깅해 AI 학습과 해석 가능성을 높임|
|**온톨로지**|도메인 개념과 관계를 구조화해 AI가 맥락을 이해하고 추론할 수 있게 함|
|**Prompt/하네스 자산화**|프롬프트, 테스트 하네스, 실행 템플릿을 재사용 가능한 AI 활용 자산으로 관리|
|**AI 평가 데이터**|AI 모델의 성능, 정확성, 안전성, 업무 적합성을 검증|
|**합성데이터 Synthetic**|부족하거나 민감한 데이터를 보완해 AI 학습과 검증에 활용|

### 한 줄 정의

> **Model-Ready Era는 데이터를 AI 모델이 이해하고, 검색하고, 생성하고, 검증할 수 있는 상태로 만드는 단계**입니다.

---

## 3단계. Agent-Ready Era

### 미래: AI Agent Era

이 단계의 핵심은 AI가 답변만 하는 것이 아니라,  
**데이터를 기반으로 판단하고, API나 Tool을 호출하고, 실행 결과를 다시 데이터로 반영하는 것**입니다.

즉, 데이터가 단순한 참고 자료가 아니라 **Agent의 실행 기반**이 됩니다.

|특화 주제|이 Era에서 중요한 이유|
|---|---|
|**API/Tool 연계 데이터**|Agent가 업무 시스템, 외부 서비스, AI 도구와 연동해 실제 행동을 수행할 수 있게 함|
|**데이터 파이프라인 Feedback Loop**|Agent의 실행 결과와 사용자 반응을 다시 데이터에 반영해 지속적으로 개선|

### 한 줄 정의

> **Agent-Ready Era는 데이터를 Agent가 읽고, 판단하고, 실행하고, 그 결과를 다시 반영할 수 있는 상태로 만드는 단계**입니다.

> 모든 AI 발전 단계에서 필요한 것은  
> **데이터를 찾고, 이해하고, 신뢰하고, 통제하고, 운영할 수 있는 기반**이다.

공통 기반 주제는 다음과 같습니다.

**데이터 카탈로그, 메타데이터, 비즈니스 Glossary, 데이터 품질 권한관리, 데이터 계통 Lineage, Observability, 데이터 운영관리, 데이터 생명주기 관리, 데이터 보안 개인·민감정보 마스킹, 데이터 Product화**

|**Agent-Ready Era**|데이터를 Agent가 직접 실행하고 개선할 수 있는 형태로 연결|API/Tool 연계 데이터, 데이터 파이프라인 Feedback Loop|

# 5. 최종 메시지 문구

> AI-Ready Data Module은 특정 한 시대의 데이터 준비가 아니라,  
> **Analysis-Ready → Model-Ready → Agent-Ready**로 진화하는 전체 데이터 준비 체계이다.
> 
> 공통적으로는 데이터의 검색성, 이해 가능성, 신뢰성, 거버넌스, 운영성이 필요하며,  
> 각 Era별로는  
> **Analysis-Ready에서는 디지털화와 전처리**,  
> **Model-Ready에서는 의미화와 평가 데이터**,  
> **Agent-Ready에서는 API/Tool 연계와 Feedback Loop**가 핵심 차별화 요소가 된다.
