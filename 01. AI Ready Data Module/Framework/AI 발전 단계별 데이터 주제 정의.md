## 전체 구조

|AI 발전 단계|시대 구분|데이터 준비 방향|핵심 질문|
|---|---|---|---|
|**1단계**|과거: Data 분석 시대|데이터를 찾고, 정리하고, 분석 가능한 형태로 만드는 단계|“사람이 데이터를 분석할 수 있는가?”|
|**2단계**|현재: AI 활용 시대|AI가 데이터를 이해하고, 학습·검색·생성에 활용할 수 있게 만드는 단계|“AI가 데이터의 의미와 맥락을 이해할 수 있는가?”|
|**3단계**|미래: AI Agent Era|Agent가 데이터를 읽고, 판단하고, 시스템과 연계해 실행할 수 있게 만드는 단계|“AI Agent가 데이터 기반으로 직접 행동할 수 있는가?”|

# 1단계. 과거 — Data 분석 시대

## Analysis-ready Data

이 단계는 데이터를 **사람과 분석 시스템이 활용할 수 있도록 정형화·표준화·관리**하는 것이 핵심입니다.  
데이터가 어디에 있고, 어떤 의미이며, 신뢰할 수 있는지 확인하는 기반을 만드는 단계입니다.

|주제|역할|
|---|---|
|**데이터 디지털화**|문서·이미지·오프라인 정보를 분석 가능한 디지털 데이터로 전환|
|**Physical 데이터**|운영 시스템·설비·현장 등 실제 원천 데이터를 확보|
|**데이터 카탈로그**|전사 데이터 자산의 위치·소유자·활용 정보를 검색 가능하게 관리|
|**메타데이터**|데이터 구조, 속성, 생성 이력, 품질 기준 등 설명 정보 제공|
|**비즈니스 Glossary**|업무 용어와 데이터 정의를 표준화하여 공통 이해 형성|
|**데이터 전처리**|분석 목적에 맞게 데이터를 정제·변환·표준화|
|**데이터 품질 권한관리**|품질 기준과 접근 권한을 관리하여 데이터 신뢰성 확보|
|**데이터 계통 Lineage**|데이터의 출처, 변환, 이동 경로, 변경 이력 추적|
|**데이터 운영관리**|데이터 공급·배포·활용 과정을 표준화하고 안정적으로 운영|
|**데이터 생명주기 관리**|데이터 생성부터 보관·활용·폐기까지 정책 기반으로 관리|

### 요약

> 1단계는 **데이터를 분석 가능한 자산으로 만드는 단계**입니다.  
> 핵심은 데이터의 **정형화, 표준화, 검색성, 품질, 관리 체계**입니다.

# 2단계. 현재 — AI 활용 시대

## AI-ready Data

이 단계는 AI가 데이터를 단순히 조회하는 것이 아니라, **의미와 맥락을 이해하고 답변·추천·생성·판단에 활용**할 수 있도록 준비하는 단계입니다.  
데이터는 사람이 보기 좋은 형태를 넘어, AI가 해석 가능한 형태가 되어야 합니다.

|주제|역할|
|---|---|
|**데이터 해설/주석**|데이터의 의미, 맥락, 정답 기준을 태깅하여 AI 학습과 해석 가능성 향상|
|**온톨로지**|도메인 개념과 관계를 구조화하여 AI의 맥락 이해와 지식 추론 지원|
|**Prompt/하네스 자산화**|프롬프트, 테스트 하네스, 실행 템플릿을 재사용 가능한 AI 활용 자산으로 관리|
|**AI 평가 데이터**|AI 모델의 정확성, 안전성, 업무 적합성을 검증하는 평가 데이터셋 제공|
|**합성데이터 Synthetic**|부족하거나 민감한 데이터를 보완하여 AI 학습·검증에 활용|
|**데이터 보안 개인·민감정보 마스킹**|AI 활용 과정에서 개인정보와 민감정보 노출을 방지|
|**데이터 Product화**|AI가 반복적으로 활용할 수 있도록 데이터셋을 재사용 가능한 상품형 자산으로 관리|
|**Observability**|AI 활용 과정에서 데이터 흐름, 이상 징후, 품질 문제를 모니터링|

### 요약

> 2단계는 **AI가 이해하고 활용할 수 있는 데이터로 고도화하는 단계**입니다.  
> 핵심은 데이터의 **의미화, 맥락화, 평가 가능성, 보안성, 재사용성**입니다.

# 3단계. 미래 — AI Agent Era

## Agent-ready Data

이 단계는 AI가 답변이나 추천에 그치지 않고, **데이터를 읽고 판단한 뒤 업무 시스템이나 도구를 호출해 직접 실행**하는 단계입니다.  
따라서 데이터는 AI가 이해할 수 있을 뿐 아니라, Agent가 행동으로 연결할 수 있어야 합니다.

|주제|역할|
|---|---|
|**API/Tool 연계 데이터**|업무 시스템, 외부 서비스, AI 도구와 연동되어 Agent가 즉시 활용할 수 있는 인터페이스 기반 데이터 제공|
|**데이터 파이프라인 Feedback Loop**|사용자 반응과 AI 실행 결과를 다시 데이터에 반영하여 지속적으로 개선|
|**데이터 Product화**|Agent가 안정적으로 사용할 수 있도록 데이터의 소유권, 품질 기준, 서비스 수준을 명확화|
|**Observability**|Agent의 데이터 사용, 실행 흐름, 이상 징후, 실패 원인을 모니터링|
|**데이터 운영관리**|Agent가 활용하는 데이터 공급·배포·운영 프로세스를 표준화|
|**데이터 품질 권한관리**|Agent가 접근 가능한 데이터와 실행 가능한 범위를 통제|
|**데이터 계통 Lineage**|Agent가 어떤 데이터 근거로 어떤 판단과 실행을 했는지 추적|
|**데이터 보안 개인·민감정보 마스킹**|Agent 실행 과정에서 민감정보 오남용과 불필요한 노출 방지|

### 요약

> 3단계는 **AI Agent가 데이터를 기반으로 직접 행동할 수 있게 만드는 단계**입니다.  
> 핵심은 데이터의 **실행 가능성, 연계성, 통제성, 추적성, 지속 개선 구조**입니다.

# 20개 주제의 단계별 주 배치

|구분|1단계 Data 분석 시대|2단계 AI 활용 시대|3단계 AI Agent Era|
|---|---|---|---|
|**Findable**|데이터 카탈로그, 메타데이터, 비즈니스 Glossary|메타데이터, 비즈니스 Glossary|데이터 카탈로그, 메타데이터|
|**Understandable**|데이터 전처리|데이터 해설/주석, 온톨로지|온톨로지|
|**Trustworthy**|데이터 품질 권한관리, 데이터 계통 Lineage|Observability, 데이터 품질 권한관리|Observability, 데이터 품질 권한관리, 데이터 계통 Lineage|

|**Evolvable**|데이터 Product화|합성데이터, AI 평가 데이터, 데이터 Product화|데이터 파이프라인 Feedback Loop, 데이터 Product화|
|**Governed**|데이터 운영관리, 데이터 생명주기 관리, 데이터 디지털화|데이터 보안 개인·민감정보 마스킹|데이터 운영관리, 데이터 생명주기 관리, 데이터 보안 개인·민감정보 마스킹|

# 최종 정리 문구

> AI 발전 단계에 따라 데이터 준비의 초점은  
> **1단계: 분석 가능한 데이터 기반 구축** →  
> **2단계: AI가 이해하고 검증할 수 있는 데이터 고도화** →  
> **3단계: Agent가 직접 실행하고 개선할 수 있는 데이터 운영 체계**로 진화한다.
