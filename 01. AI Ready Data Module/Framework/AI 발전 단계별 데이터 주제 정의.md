## 1. 핵심 메시지

> AI-Ready 데이터 체계는 특정 시점의 AI 기술만을 위한 데이터 관리 체계가 아니라,  
> AI 기술이 **분석 → 모델 활용 → Agent 실행**으로 발전함에 따라 데이터가 갖춰야 할 준비 수준을 정의하는 체계이다.
> 
> 따라서 기존 RFP상의 14개 주제를 AI 기술 발전 단계별로 재검토하고, 부족한 영역 6개를 추가하여 **20개 주제 기반의 AI-Ready Data Framework**로 재정비하였다.

기존 두산 자료는 AI-Ready Data를 Findable, Understandable, Trustworthy, Governed, Scalable 체계로 구성하고, 데이터 카탈로그·메타데이터·Glossary·전처리·해설/주석·온톨로지·Observability·권한·Lineage·생명주기·운영관리·Product화·합성데이터·Physical 데이터 등 14개 주제를 포함하고 있습니다. 이후 Framework Coverage 조정 문서에서는 과제 유형과 사용자 시나리오별 필요 Data Capability를 정의하고, 기존 Framework의 Gap을 신규 주제 또는 대주제로 반영하는 방향이 제시되어 있습니다.

# 2. AI 기술 발전에 따른 3개 Era

|AI 기술 발전 단계|시대 이름|AI의 역할|필요한 데이터 관점|
|---|---|---|---|
|**1단계**|**AI Analytics Era**|AI가 정형화된 데이터를 기반으로 분석·예측·분류 수행|**Analysis-Ready Data**|
|**2단계**|**AI Model Era**|AI 모델이 데이터의 의미와 맥락을 이해하고 검색·생성·추천 수행|**Model-Ready Data**|
|**3단계**|**AI Agent Era**|AI Agent가 데이터를 기반으로 판단하고 시스템·도구를 호출해 실행|**Agent-Ready Data**|

# 3. Era별 데이터 준비 방향

## 1단계. AI Analytics Era

### 필요한 데이터: **Analysis-Ready Data**

과거의 AI는 데이터를 분석할 수 있었지만, 스스로 데이터의 의미나 맥락을 유연하게 이해하기 어려웠습니다.  
따라서 AI가 분석하기 위해서는 데이터가 먼저 **정형화, 표준화, 전처리, 위치 정의, 품질 관리**된 상태여야 했습니다.

> **Analysis-Ready Data**는 AI가 분석할 수 있도록 데이터를 찾기 쉽고, 정리되어 있고, 신뢰 가능한 형태로 준비하는 데이터입니다.

### 이 단계에서 특히 중요한 주제

|구분|주요 주제|
|---|---|
|**Findable**|데이터 카탈로그, 메타데이터, 비즈니스 Glossary|
|**Understandable**|데이터 전처리|
|**Trustworthy**|데이터 품질 권한관리, 데이터 계통 Lineage, Observability|
|**Governed**|데이터 운영관리, 데이터 생명주기 관리, 데이터 디지털화|
|**Actionable**|Physical 데이터|

### 설명 문구

> AI Analytics Era에서는 AI가 분석할 수 있는 입력 데이터를 만드는 것이 핵심이다.  
> 이를 위해 산재된 데이터의 위치와 정의를 명확히 하고, 원천 데이터를 정제·표준화하며, 데이터 품질과 계통을 관리하는 **Analysis-Ready Data**가 필요하다.

---

## 2단계. AI Model Era

### 필요한 데이터: **Model-Ready Data**

현재의 AI는 단순히 정형 데이터를 분석하는 수준을 넘어, 문서·이미지·텍스트·업무 지식 등 다양한 데이터를 이해하고 활용합니다.  
따라서 데이터는 단순히 정리된 상태를 넘어, AI 모델이 **의미, 맥락, 관계, 정답 기준**을 이해할 수 있는 상태가 되어야 합니다.

> **Model-Ready Data**는 AI 모델이 데이터를 이해하고, 검색하고, 생성하고, 평가할 수 있도록 의미와 맥락이 부여된 데이터입니다.

### 이 단계에서 특히 중요한 주제

|구분|주요 주제|
|---|---|
|**Understandable**|데이터 해설/주석, 온톨로지|
|**Actionable**|Prompt/하네스 자산화|
|**Evolvable**|AI 평가 데이터, 합성데이터, 데이터 Product화|
|**Governed**|데이터 보안 개인·민감정보 마스킹|
|**Trustworthy**|Observability, 데이터 품질 권한관리|

### 설명 문구

> AI Model Era에서는 데이터가 AI 모델의 학습·검색·생성·평가에 활용될 수 있어야 한다.  
> 이를 위해 데이터에 해설과 주석을 부여하고, 도메인 개념과 관계를 온톨로지로 구조화하며, 프롬프트와 평가 데이터를 자산화하는 **Model-Ready Data**가 필요하다.

---

## 3단계. AI Agent Era

### 필요한 데이터: **Agent-Ready Data**

미래의 AI Agent는 데이터를 참고해 답변하는 것을 넘어, 데이터를 읽고 판단한 뒤 실제 시스템이나 도구를 호출해 행동합니다.  
따라서 데이터는 AI가 이해할 수 있는 수준을 넘어, Agent가 **실행, 검증, 추적, 개선**할 수 있는 구조로 준비되어야 합니다.

> **Agent-Ready Data**는 AI Agent가 데이터를 기반으로 판단하고, 시스템과 도구를 호출하며, 실행 결과를 다시 데이터로 반영할 수 있게 하는 데이터입니다.

### 이 단계에서 특히 중요한 주제

|구분|주요 주제|
|---|---|
|**Actionable**|API/Tool 연계 데이터, Physical 데이터|
|**Evolvable**|데이터 파이프라인 Feedback Loop, 데이터 Product화|
|**Trustworthy**|Observability, 데이터 품질 권한관리, 데이터 계통 Lineage|
|**Governed**|데이터 운영관리, 데이터 생명주기 관리, 데이터 보안 개인·민감정보 마스킹|
|**Findable**|데이터 카탈로그, 메타데이터|

### 설명 문구

> AI Agent Era에서는 데이터가 단순한 분석·학습 자산이 아니라 Agent의 실행 기반이 된다.  
> Agent가 업무 시스템, 외부 서비스, 설비, 도구를 호출하기 위해서는 API/Tool 연계 데이터, 실행 권한, Lineage, Observability, Feedback Loop를 포함한 **Agent-Ready Data**가 필요하다.

# 4. 기존 14개 주제에서 20개 주제로 확장한 이유

## 기존 RFP Framework의 한계

기존 14개 주제는 AI가 데이터를 **찾고, 이해하고, 신뢰하고, 관리**하기 위한 기본 체계로는 적절합니다.  
다만 AI 기술이 Model 활용과 Agent 실행 단계로 발전하면서 다음 영역이 충분히 설명되지 않습니다.

|부족 영역|왜 추가가 필요한가|
|---|---|
|**데이터 디지털화**|AI가 활용할 수 없는 문서·이미지·오프라인 정보를 먼저 디지털화해야 함|
|**데이터 보안 개인·민감정보 마스킹**|AI 모델과 Agent가 데이터를 활용할 때 민감정보 노출 위험이 커짐|
|**API/Tool 연계 데이터**|Agent가 실제 업무 시스템과 도구를 호출하려면 인터페이스 기반 데이터가 필요함|
|**Prompt/하네스 자산화**|프롬프트, 테스트 하네스, 실행 템플릿도 반복 활용 가능한 AI 자산으로 관리해야 함|
|**AI 평가 데이터**|AI 모델의 성능, 정확성, 안전성, 업무 적합성을 지속 검증해야 함|
|**데이터 파이프라인 Feedback Loop**|AI 결과와 사용자 반응을 다시 데이터 개선에 반영해야 함|

산출물 구성 자료에서도 기존 주제 외에 AI 평가 데이터, 개인정보·민감정보 관리, 데이터 디지털화, Feedback Loop, MCP/Tool 연결 데이터 체계 등이 추가 검토 항목으로 정리되어 있으며, 최종 체계에서는 Actionable과 Evolvable 영역으로 재배치된 주제들이 확인됩니다.

# 5. Framework 체계 재정비 방향

기존 체계는 다음과 같았습니다.

|기존 RFP 체계|한계|
|---|---|
|**Findable**|데이터 검색·탐색 중심|
|**Understandable**|AI 이해 가능성 중심|
|**Trustworthy**|신뢰성·권한·Lineage 중심|
|**Governed**|운영·생명주기·Product화 중심|
|**Scalable**|합성데이터, Physical 데이터 등 확장성 중심이나 범위가 모호함|

이를 AI 발전 단계에 맞게 아래처럼 재정비합니다.

|신규 체계|재정비 이유|
|---|---|
|**Findable**|AI가 필요한 데이터를 찾고 탐색할 수 있어야 함|
|**Understandable**|AI가 데이터의 의미와 맥락을 이해할 수 있어야 함|
|**Trustworthy**|AI가 신뢰 가능한 데이터만 활용해야 함|
|**Actionable**|AI Agent가 데이터를 기반으로 실제 행동할 수 있어야 함|
|**Evolvable**|AI 성능과 데이터 품질이 피드백을 통해 지속 개선되어야 함|
|**Governed**|데이터 활용 전 과정이 정책·보안·운영 기준에 따라 관리되어야 함|

특히 **Scalable**은 “확장 가능성”이라는 결과 중심 표현에 가까워, AI Agent 시대의 실행성과 지속 개선성을 충분히 설명하기 어렵습니다.  
따라서 기존 Scalable에 있던 **Physical 데이터**는 **Actionable**로, **합성데이터**는 **Evolvable**로 재배치하는 것이 적절합니다.

||데이터 생명주기 관리|데이터 생성부터 보관·활용·폐기까지 관리|
||데이터 디지털화|비정형·오프라인 자산을 활용 가능한 디지털 데이터로 전환|
||데이터 보안 개인·민감정보 마스킹|개인정보·민감정보 보호를 위한 보안 관리|

# 7. 장표용 결론 문구

> 기존 RFP상의 14개 주제는 AI가 데이터를 찾고, 이해하고, 신뢰하고, 관리하기 위한 기본 체계로 구성되어 있었다.  
> 그러나 AI 기술이 **분석 중심 AI → 모델 활용 AI → Agent 실행 AI**로 발전함에 따라, 데이터는 단순히 분석 가능한 상태를 넘어 모델이 이해하고 Agent가 실행할 수 있는 상태로 고도화되어야 한다.
> 
> 이에 따라 기존 주제를 AI 발전 단계별로 재검토하고, 부족한 6개 주제를 추가하여 **Findable, Understandable, Trustworthy, Actionable, Evolvable, Governed**의 6대 영역과 20개 주제로 AI-Ready Data Framework를 재정비하였다.

더 짧은 헤드 메시지는 이렇게 쓸 수 있습니다.

> **AI-Ready Data Framework는 기존 14개 데이터 관리 주제를 기반으로 하되, AI 기술 발전에 따라 요구되는 실행성·평가성·피드백·보안 요건을 보완하여 20개 주제 체계로 확장되어야 한다.**
