
#### 1. Unity Catalog 등재 기준

AI/분석 활용을 위해 Unity Catalog에 데이터를 등록할 때, 실제 고객 사례에서는 어떤 기준으로 등재 대상을 정하고 있나요?  
테이블, 컬럼 설명, 오너, 도메인, 민감도, 품질 기준 등 필수로 관리하는 메타데이터 항목 사례를 알고 싶습니다.

#### 2. 데이터 권한 관리 구성 방식

RBAC/ABAC, Row-level Security, Column Masking 등을 조합해 권한 관리를 구성한 사례가 궁금합니다.  
단순 기능 설명보다는, 역할·데이터 민감도·사용 목적별로 권한 체계를 어떻게 나누어 적용하는지 타 고객 사례 중심으로 듣고 싶습니다.

#### 3. 데이터 품질 관리 방식

Databricks 고객들은 데이터 품질 Rule을 어떤 단위로 정의하고 운영하나요?  
예를 들어 Null, 중복, 정합성, 최신성, 이상치, Drift 등을 어떤 테이블이나 파이프라인 단계에서 체크하는지 실제 적용 사례를 알고 싶습니다.

#### 4. AI/RAG용 데이터 관리 방식

문서, PDF, Chunk, Embedding, Vector Index 같은 RAG용 데이터는 Databricks 환경에서 어떤 구조로 관리하는지 궁금합니다.  
원문 데이터, 전처리 결과, Embedding, Vector Index를 각각 어떻게 저장하고 카탈로그화하는지 사례를 듣고 싶습니다.

#### 5. Lineage와 Lifecycle 관리 범위

Raw 데이터부터 정제 데이터, Feature/Embedding, AI 활용 결과까지 Lineage를 어디까지 추적하는 사례가 많은지 궁금합니다.  
또한 대용량 비정형 데이터나 Vector 데이터의 보관, 아카이빙, 삭제 정책을 Databricks에서 어떻게 운영하는지도 사례 중심으로 알고 싶습니다.
