### Pillar 1. Findable — AI가 데이터를 쉽게 찾을 수 있어야 함

|주제|내용|
|---|---|
|**데이터 카탈로그**|산재되어 있는 데이터를 한 곳에 모아 검색하고 탐색할 수 있게 해주는 데이터 통합 체계. **요구사항:** AI 학습 및 추론에 사용할 데이터 위치를 신속히 파악할 수 있어야 함|
|**메타데이터**|"데이터에 대한 데이터(Data about Data)"로서 어떤 정보를 담고 있는지, 어떤 속성과 규칙을 갖는지 알려주는 데이터 설명서. **요구사항:** 테이블·필드·값에 저장된 데이터 의미를 파악해야 함|
|**비즈니스 Glossary**|조직에서 사용하는 용어의 의미를 통일해 주는 표준 용어사전(명확히 정의된 공통 언어). **요구사항:** 동일한 용어를 서로 다른 의미로 사용하거나 여러 용어를 혼용하는 경우에 대비한 용어 기준이 필요함|

### Pillar 2. Understandable — 맥락을 잘 이해할 수 있어야 함

|주제|내용|
|---|---|
|**데이터 전처리**|AI 모델이 이해하기 적합한 형태로 데이터를 정제하고 변환. 원천 데이터에 있는 노이즈·결측치·불명확성 등을 일관된 구조로 변환. **요구사항:** 사람은 쉽게 이해할 수 있지만 AI가 인식하기 어려운 문서 양식과 포맷의 변환이 필요함|
|**데이터 해설/주석**|데이터 학습을 위한 정답 혹은 핵심 내용을 요약하여, AI가 학습·추론할 수 있는 정보를 제공 (데이터에 대한 포스트잇). **요구사항:** 많은 양의 데이터에서 핵심 요약·유형 분류를 빠르게 파악할 수 있어야 함|
|**온톨로지**|도메인 지식의 개념과 관계를 구조화하여 지식을 체계적으로 정리하고, AI 모델이 맥락을 이해하고 추론할 수 있도록 함 (비즈니스 지식의 지도). **요구사항:** 비즈니스 맥락과 인과관계에 대한 정보가 필요함|

### Pillar 3. Trustworthy — 데이터를 신뢰할 수 있어야 함

|주제|내용|
|---|---|
|**Observability**|데이터·데이터 파이프라인 및 환경을 파악할 수 있도록 지원. 현 상태와 이상 징후를 지속적으로 모니터링하여 데이터 운영 안정성 보장. **요구사항:** 데이터 품질·안전성을 모니터링할 수 있어야 함|
|**데이터 품질/권한 관리**|데이터 사용자 접근권한 통제를 통해 규정과 정책에 맞도록 보안 및 컴플라이언스를 유지. **요구사항:** 적절한 권한이 부여된 데이터 범위 내에서 학습·활용되도록 관리 필요함|
|**데이터 계통 Lineage**|데이터의 출처·변환·흐름을 추적하는 것으로 데이터의 전후를 이해할 수 있도록 보장 (End-to-end view of data flow). **요구사항:** 데이터 변경·조합 이력을 트래킹할 수 있어야 함|

### Pillar 4. Governed — 체계적으로 관리되어야 함

|주제|내용|
|---|---|
|**데이터 운영관리**|데이터 공급 전 과정을 신속한 활용을 지원하며, DevOps 통합 및 코드 기반 관리를 통해 데이터 품질과 전사적 협업을 강화. **요구사항:** 데이터 수집·정제·변환 및 학습과 추론을 위해 제공되는 전체 운영 과정이 자동화 관리되어야 함|
|**데이터 생명주기 관리**|데이터의 생성부터 폐기까지 전 생애주기를 체계적으로 관리해 효율성과 보안을 확보하고, 비용·유실 및 법적·보안 리스크를 최소화. **요구사항:** 데이터 생성 → 사용 → 폐기 전체 여정이 관리되어야 함|
|**데이터 디지털화** ⭐|수기·사진 등으로 저장되어 있는 데이터를 디지털화하고, 데이터 기록 자체를 디지털 방식으로 전환. **요구사항:** 비정형·오프라인 데이터를 AI가 활용 가능한 구조화된 디지털 자산으로 변환할 수 있어야 함|
|**데이터 보안 (민감정보 마스킹)** ⭐|AI가 활용하는 데이터에 포함된 개인정보·민감정보를 식별하고 마스킹·익명화 처리하여 안전하게 활용. **요구사항:** 프라이버시 보호와 AI 활용 가능성을 동시에 확보할 수 있어야 함|

### Pillar 5. Evolvable — 지속적으로 진화할 수 있어야 함

|주제|내용|
|---|---|
|**데이터 Product화**|데이터를 누구나 즉시 사용할 수 있는 상품으로 제공해 활용성과 재사용을 높이고, 데이터에 대한 명확한 책임과 서비스 수준으로 신뢰를 보장. **요구사항:** 데이터 오너/사용자가 쉽게 데이터를 공유하고 활용할 수 있는 체계가 필요함|
|**합성데이터 (Synthetic)** ⭐|실제 데이터를 대체하거나 보완하기 위해 AI 모델이 생성한 인공 데이터로, 데이터 부족·편향·프라이버시 문제를 해결. **요구사항:** 실데이터를 대체할 수 있는 수준의 품질과 다양성을 갖춘 합성 데이터를 생성·관리할 수 있어야 함|
|**AI 평가 데이터** ⭐|AI 모델의 수행 결과를 측정하고 검증하기 위한 정답 데이터·평가 기준·벤치마크 데이터셋. **요구사항:** 모델 성능의 정량적·정성적 평가가 가능한 데이터 체계가 갖춰져야 함|
|**데이터 파이프라인 Feedback Loop** ⭐|Agent 실행 과정에서 발생하는 로그·결과·오류 데이터를 수집하여 모델 개선과 프로세스 고도화에 환류하는 체계. **요구사항:** 실운영 데이터가 지속적으로 학습·개선 사이클에 반영될 수 있어야 함|

### Pillar 6. Actionable — 즉시 행동으로 연결될 수 있어야 함

|주제|내용|
|---|---|
|**Physical 데이터** ⭐|센서·IoT·설비 등 현실 세계의 물리적 환경에서 수집되는 데이터로, AI의 실시간 판단과 실행을 지원. **요구사항:** 오프라인·현장 데이터를 AI가 즉시 활용할 수 있는 형태로 수집·연결할 수 있어야 함|
|**API/Tool 연계 데이터** ⭐|API·Tool이 수행하는 기능과 입출력을 데이터적으로 정의하고 표준화하여 AI가 호출·활용할 수 있도록 구조화 (MCP화 등). **요구사항:** Tool의 행동 범위와 결과를 AI가 이해하고 활용할 수 있는 형태로 등록·관리되어야 함|
|**Prompt/Harness 자산화** ⭐|사람의 업무 방식·판단 기준·절차를 Agent에게 지시할 수 있는 Prompt·Harness 형태로 구조화하고 버전 관리. **요구사항:** 업무별 Prompt가 재사용 가능한 자산으로 축적되고 지속적으로 개선될 수 있어야 함|
