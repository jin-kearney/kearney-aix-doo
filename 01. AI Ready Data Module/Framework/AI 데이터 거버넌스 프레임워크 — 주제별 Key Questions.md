### Pillar 1. Findable

**데이터 카탈로그**

- 여러 시스템에 분산된 데이터를 하나의 카탈로그로 어떻게 통합·등록할 것인가?
- AI 모델이 필요한 데이터를 자동으로 탐색하고 연결할 수 있도록 어떻게 구조화할 것인가?
- 카탈로그를 최신 상태로 유지하는 자동화 체계를 어떻게 만들 것인가?

**메타데이터**

- 테이블·필드명만 봐서는 의미를 알 수 없는 데이터를 어떻게 체계적으로 설명할 것인가?
- AI가 데이터의 의미와 맥락을 스스로 파악할 수 있도록 메타데이터를 어떻게 설계할 것인가?
- 데이터가 추가·변경될 때 메타데이터를 어떻게 자동으로 갱신할 것인가?

**비즈니스 Glossary**

- 부서마다 다르게 쓰이는 용어를 어떻게 하나의 기준으로 통일할 것인가?
- 표준 용어 정의를 AI 모델이 이해하고 활용할 수 있는 형태로 어떻게 연결할 것인가?
- 비즈니스 변화에 따라 용어 정의가 바뀔 때 어떻게 일관되게 관리할 것인가?

---

### Pillar 2. Understandable

**데이터 전처리**

- PDF·표·이미지 등 AI가 바로 읽지 못하는 비정형 데이터를 어떻게 구조화된 형태로 변환할 것인가?
- 결측치·오류·중복 등 데이터 품질 문제를 어떻게 일관된 기준으로 처리할 것인가?
- 원천 데이터가 바뀌어도 전처리 결과물의 품질을 어떻게 안정적으로 유지할 것인가?

**데이터 해설/주석**

- 대량의 데이터에 정확한 라벨·요약·분류를 어떻게 효율적으로 부여할 것인가?
- 사람마다 다르게 해석될 수 있는 데이터를 어떻게 일관된 기준으로 주석 처리할 것인가?
- AI가 생성한 주석의 품질을 어떻게 검증하고 사람의 검토 범위를 어떻게 설정할 것인가?

**온톨로지**

- 문서나 데이터에 흩어져 있는 비즈니스 지식과 인과관계를 어떻게 구조화할 것인가?
- AI가 단순 키워드가 아닌 개념 간의 관계를 이해할 수 있도록 어떻게 지식 체계를 설계할 것인가?
- 비즈니스 변화에 따라 온톨로지를 어떻게 지속적으로 업데이트할 것인가?

### Pillar 3. Trustworthy

**Observability**

- 데이터 파이프라인 어느 단계에서 문제가 생겼는지 어떻게 빠르게 탐지할 것인가?
- 데이터 품질 이상을 사람이 개입하지 않아도 자동으로 감지하는 체계를 어떻게 구성할 것인가?
- AI 추론 결과가 갑자기 달라졌을 때 데이터 문제인지 모델 문제인지 어떻게 구분할 것인가?

**데이터 품질/권한 관리**

- AI 학습에 사용하면 안 되는 데이터를 어떻게 사전에 걸러낼 것인가?
- 데이터 접근 권한을 역할·목적에 따라 어떻게 세밀하게 통제할 것인가?
- 데이터 품질 기준을 어떻게 정의하고 자동으로 측정·관리할 것인가?

**데이터 계통 Lineage**

- AI 결과에 문제가 생겼을 때 어떤 원천 데이터가 원인인지 어떻게 역추적할 것인가?
- 데이터가 어떤 변환·조합 과정을 거쳤는지 어떻게 자동으로 기록할 것인가?
- 규제·감사 대응 시 데이터 출처와 흐름을 어떻게 즉시 증명할 것인가?

---

### Pillar 4. Governed

**데이터 운영관리**

- 데이터 수집·정제·변환·제공까지의 전체 파이프라인을 어떻게 자동화할 것인가?
- 파이프라인 오류 발생 시 빠르게 복구하고 영향 범위를 어떻게 최소화할 것인가?
- 여러 팀이 동시에 데이터를 활용할 때 충돌 없이 어떻게 협업 체계를 유지할 것인가?

**데이터 생명주기 관리**

- 더 이상 유효하지 않은 데이터를 어떻게 식별하고 적시에 폐기·이관할 것인가?
- 법적 보존 의무가 있는 데이터와 폐기 가능한 데이터를 어떻게 자동으로 분류·관리할 것인가?
- 데이터가 쌓일수록 늘어나는 저장·관리 비용을 어떻게 생명주기 정책으로 통제할 것인가?

**데이터 디지털화** ⭐

- 수기·사진·종이 형태의 데이터를 AI가 바로 활용할 수 있는 구조로 어떻게 전환할 것인가?
- 디지털화 과정에서 발생하는 오류·누락을 어떻게 검증하고 품질을 보장할 것인가?
- 현장 업무 프로세스를 바꾸어 데이터가 처음부터 디지털로 생성되도록 어떻게 설계할 것인가?

**데이터 보안 (민감정보 마스킹)** ⭐

- 비정형 데이터(문서·대화·로그)에 숨어 있는 개인정보·민감정보를 어떻게 자동으로 탐지할 것인가?
- 마스킹·익명화 처리 후에도 AI 학습·추론 성능이 저하되지 않도록 어떻게 균형을 맞출 것인가?
- 데이터 활용 목적과 규제 요건에 따라 마스킹 수준을 어떻게 차등 적용할 것인가?

### Pillar 5. Evolvable

**데이터 Product화**

- 데이터를 요청 없이도 누구나 찾아 바로 쓸 수 있는 셀프서비스 체계를 어떻게 만들 것인가?
- 데이터셋별 품질·갱신 주기·책임자를 어떻게 명확히 정의하고 유지할 것인가?
- 데이터 재사용률을 높이고 중복 생산을 줄이기 위한 공유 체계를 어떻게 설계할 것인가?

**합성데이터 (Synthetic)** ⭐

- 실데이터가 부족하거나 사용이 제한된 영역에서 AI 학습용 데이터를 어떻게 합성·생성할 것인가?
- 합성 데이터가 실제 데이터의 패턴과 분포를 충분히 반영하는지 어떻게 검증할 것인가?
- 합성 데이터 사용 시 발생할 수 있는 편향이나 과적합을 어떻게 방지할 것인가?

**AI 평가 데이터** ⭐

- AI 결과물의 좋고 나쁨을 판단하는 평가 기준과 정답 데이터를 어떻게 정의하고 구축할 것인가?
- 도메인 전문가의 판단을 평가 데이터로 어떻게 효율적으로 수집·축적할 것인가?
- 모델이 업데이트되거나 업무가 바뀔 때 평가 데이터를 어떻게 지속적으로 갱신할 것인가?

**데이터 파이프라인 Feedback Loop** ⭐

- Agent가 실제 수행하면서 발생한 결과·오류 데이터를 어떻게 자동으로 수집하고 저장할 것인가?
- 실운영 데이터를 모델 재학습과 프롬프트 개선에 어떻게 체계적으로 반영할 것인가?
- 피드백 반영 주기와 자동화 수준을 업무 특성에 맞게 어떻게 설계할 것인가?

### Pillar 6. Actionable

**Physical 데이터** ⭐

- 현장·설비·센서 데이터를 AI가 실시간으로 활용할 수 있도록 어떻게 수집·연결할 것인가?
- 노이즈·누락·이상값이 많은 물리 데이터를 AI 입력에 적합한 형태로 어떻게 정제할 것인가?
- 실시간 처리가 필요한 데이터와 배치로 처리해도 되는 데이터를 어떻게 구분하여 설계할 것인가?

**API/Tool 연계 데이터** ⭐

- 각 API·Tool이 수행하는 기능과 입출력 스키마를 AI가 이해할 수 있는 형태로 어떻게 표준화할 것인가?
- 수많은 Tool을 MCP 등 공통 프로토콜로 어떻게 등록하고 일관되게 관리할 것인가?
- Tool 호출 이력과 결과를 어떻게 저장하여 재사용 가능한 데이터 자산으로 축적할 것인가?

**Prompt/Harness 자산화** ⭐

- 사람의 업무 방식과 판단 기준을 Agent가 실행할 수 있는 Prompt 형태로 어떻게 구조화할 것인가?
- 잘 작동하는 Prompt를 버전으로 관리하고 조직 전체가 재사용할 수 있도록 어떻게 자산화할 것인가?
- Prompt 성능 저하나 업무 변경이 생겼을 때 어떻게 빠르게 탐지하고 개선할 것인가?
