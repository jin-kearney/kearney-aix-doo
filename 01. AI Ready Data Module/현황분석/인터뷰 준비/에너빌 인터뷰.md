
| 단계         | 확인하고 싶은 내용       | 질문 예시                                                        |
| ---------- | ---------------- | ------------------------------------------------------------ |
| **데이터 수집** | 필요한 데이터 식별 가능 여부 | AI 과제 시작 시 필요한 데이터가 무엇인지 명확히 정의할 수 있었나요?                     |
|            | 데이터 위치·보유 주체 파악  | 필요한 데이터가 어느 시스템, 어느 부서에 있는지 쉽게 찾을 수 있었나요?                    |
|            | 데이터 확보 방식        | 실제 데이터는 어떤 방식으로 확보했나요? 시스템 연계, 수기 요청, Excel 전달 등 어떤 방식이었나요?  |
|            | 수기·비정형 데이터 존재 여부 | 시스템에 없고 담당자가 별도로 관리하는 Excel, 문서, 메일 데이터가 있었나요?               |
|            | 데이터 접근 병목        | 데이터 요청, 승인, 추출, 전달 과정에서 시간이 오래 걸린 부분은 무엇이었나요?                |
| **데이터 관리** | 데이터 의미 이해        | 데이터 항목의 의미, 기준, 단위, 생성 로직을 이해하는 데 어려움이 있었나요?                 |
|            | 데이터 품질 이슈        | 결측, 오류, 중복, 최신성, 정합성 문제로 인해 활용이 어려웠던 데이터가 있었나요?              |
|            | 데이터 기준 불일치       | 부서나 시스템마다 같은 용어·지표를 다르게 정의해 혼선이 있었나요?                        |
|            | 권한·보안 이슈         | 개인정보, 민감정보, 권한 문제로 데이터 활용이 지연되거나 제한된 적이 있나요?                 |
|            | 변경 이력·출처 추적      | 데이터가 어디서 생성되고 어떻게 가공되었는지 추적할 수 있었나요?                         |
| **데이터 활용** | AI 활용 가능 형태 여부   | 확보한 데이터가 모델 개발이나 분석에 바로 사용할 수 있는 형태였나요?                      |
|            | 전처리 부담           | AI 과제 수행 전에 정제, 변환, 라벨링, 통합 작업이 얼마나 필요했나요?                   |
|            | 학습·평가 데이터 준비     | 학습 데이터와 평가 데이터를 구분하거나 품질을 검증하는 기준이 있었나요?                     |
|            | 재사용 가능성          | 과제에서 만든 데이터셋, 전처리 로직, Feature, Prompt 등을 이후 과제에 재사용할 수 있었나요? |
|            | 운영 반영 가능성        | PoC 이후 실제 운영 환경에서 데이터를 지속적으로 공급받을 수 있었나요?                    |
|            | 성과 영향            | 데이터 문제로 인해 일정 지연, 모델 성능 저하, 추가 비용, 과제 범위 축소가 발생했나요?          |
