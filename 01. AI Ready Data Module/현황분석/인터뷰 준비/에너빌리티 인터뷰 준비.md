# 인터뷰 세션 구성안

## Part 1. 사전질의서 기반 AI Ready Data Framework 검증
### 목적

사전질의서 답변에서 나온 문제를 하나씩 쪼개고, 각 문제를 **AI Ready Data 20개 주제 중 가장 직접적인 1개 주제**에만 매핑
## 데이터 주제로 다룰 이슈만 매핑

> 질문 방향:  
> “이 문제를 AI Ready Data Framework의 어떤 주제에서 다룰 것인가?”  
> “해당 주제의 Key Question으로 충분히 커버되는가?”

| 건설현장 공종 단위와 제작공장 PO 단위 데이터 연결이 어려움        | 서로 다른 업무 개념 간 관계를 구조화할 수 있는가?              | **B-3 온톨로지**            |
| 설계도서, 규격, 업무 데이터 간 연결고리가 부족함              | 문서·데이터·업무 개념 간 관계를 정의할 수 있는가?              | **B-3 온톨로지**            |
| 운영 중 데이터 품질 저하나 모델 성능 변화를 감지하지 못함         | 운영 중 데이터 이상, 결측 증가, 분포 변화를 감지할 수 있는가?      | **C-1 Observability**   |
| 본문에 “첨부 참조”만 있고 실제 텍스트가 없는 경우가 많음         | AI 활용 전 데이터 완전성과 사용 가능성을 판정할 수 있는가?        | **C-2 데이터 품질 권한관리**     |
| 데이터 출처, 버전, 전처리 이력을 별도 관리하지 않음            | 학습 데이터의 출처와 변환 이력을 추적할 수 있는가?              | **C-3 데이터 계통 Lineage**  |
| API 부재, SQL 제한으로 CSV 다운로드나 RPA에 의존함       | AI가 활용할 수 있는 시스템 API/Tool/I/F를 표준화할 수 있는가? | **D-2 API/Tool 연계 데이터** |
| RAG/MCP 서버를 통해 그룹 데이터를 호출하고 싶음            | AI Agent가 데이터를 안전하게 호출할 수 있는 Tool 체계가 있는가? | **D-2 API/Tool 연계 데이터** |
| 반복 과제에서 산출된 데이터셋이나 결과물이 재사용되지 않음          | 데이터를 반복 재사용 가능한 Product 단위로 관리할 수 있는가?     | **E-1 데이터 Product화**    |
| 데이터 확장 요청, 표준화, 적재, 재배포 프로세스가 필요함         | 데이터 생성부터 재사용까지 생애주기 기준이 있는가?               | **F-2 데이터 생애주기 관리**     |
| Excel·개인 파일이 시스템 데이터보다 최신·정확한 경우가 있음      | 산재된 Excel·개인 파일을 공식 데이터 자산으로 전환할 수 있는가?    | **F-3 데이터 디지털화**        |
