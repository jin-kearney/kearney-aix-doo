# Part 1. 사전질의서 기반 AI Ready Data Framework 검증

**사전질의서 답변을 기반으로 현황 이슈를 유형화**하고, 각 이슈를 AI Ready Data Framework의 데이터 주제와 연결하여 검증

> “사전질의서에서 드러난 데이터 이슈가 AI Ready Data 20개 주제에서 충분히 다뤄지고 있는가?”  
> “각 주제의 Key Question이 실제 현장의 문제를 설명하고 해결 방향을 제시할 수 있는가?”

예를들어, 

| 사전질의서 답변 내용                                                                                     | 공통 이슈 유형                        | 매핑 주제                 | Framework Key Question                                        |
| ----------------------------------------------------------------------------------------------- | ------------------------------- | --------------------- | ------------------------------------------------------------- |
| 정성문 수석: 타 조직 데이터 위치·관리 주체를 매번 수소문  <br>이성민 수석: 비정형 데이터는 현업에 별도 요청  <br>김용화 수석: 데이터 관리 자체가 되지 않음 | 필요한 데이터의 위치와 보유 주체를 찾기 어려움      | **A-1 데이터 카탈로그**      | AI 과제에 필요한 데이터 자산의 존재, 위치, 오너, 접근 경로를 사전에 찾을 수 있는가?           |
| 정성문 수석: 데이터 정의가 개발 중 변경됨  <br>이성민 수석: 설비 운전 데이터 분석에 도메인 지식 필요  <br>김용화 수석: 메타데이터 불일치, 단위 불일치 발생 | 데이터의 의미, 단위, 생성 기준, 갱신 기준이 불명확함 | **A-2 메타데이터**         | AI가 데이터 구조, 단위, 생성 기준, 갱신 주기, 기준일을 이해할 수 있도록 메타데이터가 관리되고 있는가? |
| 정성문 수석: 담당자별 데이터 해석 차이  <br>이성민 수석: 도메인 지식 없이는 설비 데이터 해석 어려움  <br>김용화 수석: 현업 담당자 간 도메인 지식 차이 큼  | 조직·담당자별 용어와 해석이 다름              | **A-3 비즈니스 Glossary** | 동일 용어, 지표, 코드, 현장 표현을 조직 간 같은 의미로 해석할 수 있도록 표준 정의되어 있는가?      |

사전질의서에서 이미 확인된 이슈 외에, 인터뷰에서는 6 Pillar 기준으로 아래 내용을 추가 확인합니다.

|Pillar|추가 확인 질문|
|---|---|
|**Findable**|AI 과제에서 반복적으로 찾게 되는 핵심 데이터는 무엇이며, 카탈로그를 만든다면 어떤 데이터부터 등록해야 하는가?|
|**Understandable**|반드시 표준 정의가 필요한 용어, 코드, 지표, 식별자는 무엇이며, 담당자 간 해석 차이는 어떻게 조정되어야 하는가?|

# Part 2. AI 과제 실행 경험 기반 사례 분석

실제 각자가 수행한 AI 과제 사례를 중심으로, 데이터 관련 Pain Point가 **수집 > 관리 > 활용** 단계에서 어떻게 발생했는지 확인하고자 합니다 **(Why AI Ready Data 사례 파악)**

> “실제 AI 과제에서 데이터 문제는 어느 단계에서 가장 크게 발생했는가?”  
> “수집, 관리, 활용 단계별 병목은 무엇이었고, 일정·비용·성능에 어떤 영향을 주었는가?”


# Part 3. 데이터 관점 조직·거버넌스 현황 분석

사전질의서 내용을 기반으로 조직.거버넌스를 유형화하고, **추가 질문을 통해**
데이터가 어떤 조직을 거치고 있는지와 거버넌스 현황을 파악하고자 합니다. 

> “데이터 관점에서 조직적 문제는 무엇인가?”  
> “데이터 오너십, 표준, 품질, 보안, Function별 Silo 문제는 현재 어떻게 관리되고 있는가?”
