□	두산지주 CHRO 현업 인터뷰
	(커니) 현재 진행 중인 AI과제는 어떤 것인지?
	(배혜련 팀장) 현재 HR 분야 안에서 업무 효율성 및 생산성 향상을 위한 AI 과제를 도출한 상황임. 데이터 얘기를 한 이유는 직접 PoC를 raw code, no code, vibe code를 통해서 진행 중임. 실질적으로 시스템을 구축하는 것은 DE센터와 같이 할 것인데 여러 개의 과제 풀 중에서 CHRO가 몇 개 골라서 해보는 중임. 하지만 AI를 만들기 위해서 어떤 데이터가 필요한지 자체는 도메인 지식이 있기 때문에 알지만, 인사 데이터는 내부적으로 권한이 나뉘어 있어서 민감 정보가 다수 존재함. 그래서 PoC를 진행하기 위해서 더미 데이터를 사용해도 되지만 사장님이 실제 데이터로 진행하면 좋겠다는 의견을 냄. 하지만 비개발자기 때문에 비식별화하는 방법을 모름. 
	(배혜련 팀장) 또한 비식별화의 수준이 어느 정도여야 하는지도 SA팀을 통해 확인을 계속하고 있고 전처리와 AI가 읽기 쉽게 하는 데이터로 만드는 것에 대한 지식이 없어서 과제를 같이 하고 있는 컨설팅사의 도움을 많이 받는 중임. 이런 과정 중에, 처음에 생각했던 필요한 데이터의 범위가 계속해서 늘어나는 중임. 이 늘어나는 것들에 대해서도 언제까지 데이터를 확인해야 하는지도 불분명함. 현재 돌리고 있는 파일럿을 다른 function의 도메인 지식을 가진 인원이 앞으로 PoC를 진행할 텐데 이들이 개발 조직이랑 원활하게 커뮤니케이션할 수 있고 더 나아가서 도메인 개발자가 될 수 있어야 한다는 관점으로 진행 중임.
	(커니) 현재 진행 중인 파일럿 프로젝트는 몇 개 정도인지? 
	(배혜련 팀장) Raw 코드가 8개, Vibe 코드가 4개로 총 12개이며, 서브 프로젝트로 3개 정도 추가될 예정임. PoC로 진행하는 것이기 때문에 7~8월 정도에 끝낼 예정임
	(커니) 민감정보 마스킹 외에 AI 과제 진행 중 데이터 관점의 pain point는 없는지?
	(배혜련 팀장) PoC를 진행 시 1차 수집 이후에 계속해서 필요한 데이터가 생기고 이를 수집함.
	(조욱희 수석) 데이터 준비와 프로세스를 모르는 상태에서 일단 시작했음. AI를 어떻게 설계할지 먼저 그려놓고 거기에 맞는 데이터를 찾아나가는 방식으로 하다 보니 시간이 오래 걸리고 방향이 바뀌게 되는 과정이 있었음. 데이터 전처리 과정이 어디서부터 어디까지 어떻게 준비해야 하는지에 대한 감이 없었음.
	(커니) AI과제를 진행하기 위해서 어떤 데이터를 봐야 되는지 정해져 있던 상황인지?
	(조욱희 수석) 어떤 데이터를 봐야 하는지 이제 찾아가는 과정이었고 그 과정에서 데이터가 계속해서 추가되는 것임. 그나마 인사는 그래도 시스템화되어 있기 때문에 SAP 등을 통해서 웬만한 것들은 내려받을 수 있음. 하지만 기본적으로 내려받을 수 있다고 생각했던 데이터가 없다거나 못 찾겠기에 DDI에 요청하면 DDI에서도 데이터가 조합돼서 나와야 하므로 바로 뽑아줄 수 없는 상황이 생김.
	(커니) AI과제에 활용하기 위한 데이터를 찾던 중 원래 보던 데이터와 raw 데이터가 달랐던 이슈가 있었는지?
	(조욱희 수석) 예를 들어 보상이라고 하면 연봉 등급, 연봉 인상률 들의 연봉 정보가 결과적으로 과거 데이터가 다 시스템에 업로드돼서 개인별로 피드백함. 피드백했다는 것은 어딘가에 데이터가 있다는 것인데 한 번에 원하는 데이터를 내려받지 못하거나 수식을 걸어야만 나오는 데이터가 있음.
	(커니) ERD를 사용해서 어디에 어떤 데이터가 있는지 대략적으로 알 수 있을 것 같은데 아닌지?
	(배혜련 팀장) ERD가 무엇인지 모름.
	(유지인 수석) 과제에 보상 모듈과 평가 모듈이 있고 모듈원은 데이터에 대한 정보 등을 잘 알고 있는데 그 과제에서 어느 정도의 scope를 할 것인지를 모듈원끼리 정함. 그래서 정한 scope를 갖고 필요한 데이터를 정했는데 더 필요한 데이터가 생기게 됨. 평소에 본인은 시스템에 있던 데이터를 사용했지만, 추가로 안 쓰던 DB에서 데이터를 확보해야 하니 낯설기도 해서 필요한 데이터를 정확하게 받지 못함. 이런 것들이 준비된 상태로 과제가 진행돼야 하지만 데이터를 보완하는 과정이 계속해서 병행 및 반복되고 있음.
	(커니) 현재 HR에서 보고 있는 데이터는 SAP가 기준인지?
	(조욱희 수석) 정형 데이터는 SAP 위주로 보고 있음. PPT 같은 비정형 데이터를 AI가 사용할 수 있게 전환하는 방법이나 도구에 대한 지식이 없어서 컨설팅사에 의존하고 있음. 이 과정에서 외부 tool을 사용하는 것은 보안 측면에서 막혀있기 때문에 내부에서 비정형 데이터를 파싱해야 하고 조직도 같은 경우는 내부 시스템에 있는 조직 체계와 연동돼야 하는데 일일이 수작업으로 할 수밖에 없음.
	(커니) 조직 체계는 인사 시스템에 올라와 있는 조직도를 사용하면 되는 것이 아닌지?
	(조욱희 수석) 기본적으로 조직도가 조직명이 있고 조직명에 맞는 코드가 지정돼 있을 텐데 CHRO에서 만드는 조직도 체계 보고 자료에는 코드가 지정 안돼 있고 실질적으로 full name은 있는데 이를 축약해서 쓰기도 함. 그래서 데이터 준비 작업이 안 돼 있는 것임.
	(커니) 시스템 조직도가 원본으로 존재하는 것이 아닌지?
	(배혜련 팀장) 시스템 조직도가 없다고 보면 됨. Hierarchy는 있는데 이를 도식화해서 보이게끔 되어 있는 것이 없음. 이를 보기 쉽게 하려고 과거부터 사용한 보고 프레임이 있는데 그냥 조직도 프레임에 수작업으로 넣어야 함. 조직 정보의 원천 자체는 시스템에 있음.
	(커니) 비정형 PPT 조직도를 사용하는 이유가 무엇인지?
	(배혜련 팀장) 이를 보고해서 의사결정을 받기 때문에 조직 정보를 도식화하는 것임. SAP에서는 포지션 생성을 하고 그 값이 남아 있는 것이고 모든 보고는 수작업으로 된 조직도 도식을 반영한 것임.
	(표솔잎 수석) 시스템은 반영 후가 나와야 되는 것이고 반영 전에는 손으로 그려서 보고를 하는 것임.
	(커니) 추후에 이를 사용하기 위해서 반영 후 시스템에 남아 있는 것을 쓰면 되는 것이 아닌지?
	(배혜련 팀장) CHRO에서 진행 중인 15개 과제 중의 하나인데 수작업으로 시스템에 발령 내는 것을, 조직도를 넣으면 자동으로 시스템에 등록되는 것을 만들고 싶은 것임. 반대로 시스템에 나와 있는 조직 정보를 도식화해서 나오게 할 수도 있겠지만 일단은 그렇게 진행 중임. 조직도는 아닌데 이런 반대 방향으로 나오게 하는 것이 있긴 있음. Successor라고 포지션에 맞는 사람들을 보여주는 것이 있음. SAP가 만들었는데 보기 좋지 않기 때문에 어차피 내려받아서 다시 수정해야 함.
	(커니) 조직도를 기반으로 자동으로 발령하는 시스템 과제 외에 다른 비정형 데이터가 있는지?
	(배혜련 팀장) 많음. 예를 들어서 취업 규칙 같은 것들은 전부 웹상에 올라가는 워드 파일임.
	(유지인 수석) 교육 운영 관련된 교육 과정 안내, 교육 참석자 체크 같은 프로세스가 메일이나 PPT, 워드 등으로 진행됨. 
	(조욱희 수석) 이메일처럼 개인 정보 데이터가 많아서 비식별화하는 것이 불가능한 형태도 있음. 엑셀은 코드 기반으로 비식별 처리할 수 있을 것 같은데 이메일은 방법을 모름. 예를 들어 PoC 환경에서 진행할 때 랜딩 존안에서 진행하면 어느 정도 보안이 되겠지만 랜딩 존 안에 있는 툴 자체가 기능이 많이 없고 외부 API를 쓰기 위해서는 일일이 가명 처리해야 함. 
	(커니) 비식별 처리에 레벨이 있을 것 같은데 맞는지?
	(조욱희 수석) 맞음. 프로젝트마다 조직명까지 가명 처리를 해야 하는 것이 있고 아닌 것도 있음. 필수적으로 사람 이름, 주민번호, 생년월일 같은 정보는 가명 처리하고 있음.
	(커니) 가명 처리는 툴을 사용하고 있는지?
	(조욱희 수석) 프로젝트의 성격이 한 팀에서 권한 있는 사람만 참여하는 것이 아니라 계열사끼리 보면 안 되는 정보가 있기 때문에 예를 들어 회사별로 보상 권한이 있는 사람이 참여하는 것임. 전체 데이터에 대한 권한을 가진 사람은 없음. 또한 랜딩 존이 아닌 외부에 갔다 오는 데이터는 커버가 안 되고 가명 처리됐다가 돌아올 때 풀리는 방법이 없기 대문에 작업하기 어려움. 모든 프로젝트 과제마다 가명 처리 항목과 방법을 정의해야 함.
	(유지인 수석) 12개의 과제를 CHRO에서 PM 조직으로서 PwC 컨설팅의 인원들과 함께 필요 데이터 정의부터 수집까지 일괄적으로 진행하고 있고 12개의 모듈을 각자 서버에 올리기 때문에 이런 어려움이 더 크게 느껴졌던 것 같음. 
	(커니) 12개의 과제 진행에서 데이터 취합 등의 과정을 CHRO에서 직접 진행하는 것인지?
	(유지인 수석) 모듈 중에서 직접 참여하는 모듈은 직접 데이터를 수집한 것도 있고 직접 참여하지 않는 모듈은 데이터 수집 리스트와 필요성을 논의하고 수집에 도움을 줌. 어쨌든 모든 데이터가 CHRO에 모여서 비식별화 등이 진행됨.
	(두산지주) 비식별화 같은 것은 툴을 사용하는지?
	(유지인 수석) 정형 데이터는 필드가 다 정의돼 있어서 컨설턴트가 메타 데이터를 정리해서 어떤 방식으로 비식별화하겠다는 것을 SA팀과 협의함. 그 협의가 이뤄진 항목, 로직, 툴에 따라 코딩하고 그 코드를 베이스로 CHRO에서 원본 데이터를 외부로 나가기 전에 두산 CHRO PC에서 비식별화함. 이 결과를 서버에 업로드하는 형태로 진행 중임. 한 땀 한 땀 돌려서 진행함.
	(조욱희 수석) 원본이 없기 때문에 오류가 많이 발생함.
	(조욱희 수석) PoC에서 진행할 때 쓰이는 비정형 데이터가 그렇게 많지 않기 때문에 수기로 삭제 혹은 가명 처리하고 있음.
	(커니) 비정형 데이터의 관리 주체 등을 알고 있었기 때문에 데이터를 찾을 수 있었는지?
	(조욱희 수석) 과제 범위에 필요한 데이터를 식별한 다음에 전달하는 방식으로 해서 CHRO에서 보고 가명 처리할지 등을 판단하고 있음.
	(커니) 현재 커니에서 진행 중인 과제 중에 데이터 카탈로그가 있는데 여기에 인사 데이터를 올리게 되면 그것 자체도 보안 이슈가 있을 것 같은데 맞는지?
	(배혜련 팀장) 맞음. 어떤 자료가 있다는 것도 몰라야 할 수도 있음. 그래서 어떤 회사의 HR은 원드라이브도 사용 안 하고 원드라이브를 쓰더라도 폴더에 제한된 사람만 접근할 수 있게 설정함. 같은 HR 안에서도 사람마다 권한이 다름.
	(커니) 현재 폴더 같은 것에 대한 권한 관리는 어떻게 되는지?
	(배혜련 팀장) 두산지주는 폴더를 관리하는 사람이 1명 있어서 그 사람이 권한을 넣고 빼고를 결정함. 보통은 팀에서 프로젝트 폴더와 CoE폴더가 있고 대부분 팀 폴더로 운영하고 있음. 팀 폴더는 팀 안에서는 다 볼 수 있고 프로젝트 폴더 같은 것은 권한이 있는 사람만 볼 수 있도록 권한 설정을 해 놓았음.
	(유지인 수석) 팀즈 원드라이브를 통해서 관리하기 때문에 수동으로 관리하고 있음.
	(두산지주) 그럼 보상 관련 데이터 빼고는 다른 계열사 HR에 오픈할 수 있는 것이 아닌지?
	(배혜련 팀장) 지주는 자회사의 HR 컨설팅을 하는 역할을 수행하는 것이기 때문에 다 볼 수 있지만 각 계열사는 계열사의 것만 봐야 하고 HRM과 HRD가 있어서 서로 관련된 것을 보면 안됨.
	(커니) 파일을 폴더에 잘못 올리면 그걸 보지 말아야 할 사람이 볼 수도 있는데 관련 이슈가 있었는지?
	(배혜련 팀장) 아직까지는 없었음. 그런 리스크가 있는 파일은 애초에 원드라이브에 안 올림. 로컬에서 관리하거나 그 팀만 볼 수 있는 곳에 올림.
	(유지인 수석) 프로젝트를 하면서 특정 파일을 공유할 때 화이트리스트를 설정해서 특정 사람만 볼 수 있게 할 수도 있는데 기본적으로 팀 단위로 폴더가 관리되고 팀 내에서도 채널 단위로 관리가 되기 때문에 채널 안에 있는 폴더나 파일은 따로 관리하지 않음.
	(커니) PoC과제 진행 전체 기간 중에서 데이터 준비에 걸리는 시간은 어느 정도인지?
	(유지인 수석) 기본적으로 no 코드, raw 코드는 5주를 기준으로 생각하고 있고 데이터 식별 및 준비 기간을 2주로 고려했음. 하지만 현재 2~3 주차인데 아직도 데이터에 보완할 게 생김. 예를 들어 데이터가 보완할 게 생겨서 로컬에 있는 것을 바로 모듈에 업로드할 수 있는 것이 아니라 CHRO에서 검토 및 비식별화를 해야 하므로 시간이 소요됨. 그렇게 되면 3주 차에는 그 데이터를 바로 쓰지 못하고 그다음 주에 제대로 업로드되었는지 등을 확인해야 진행할 수 있음.
	(커니) 현재 12개 과제 중에서 일부만 PoC를 진행 중인 것인지?
	(유지인 수석) 12개를 동시에 진행 중임.
	(배혜련 팀장) 자회사가 들어와서 과제별로 한 6~7명씩 있고 비개발자들이 멘토링 받으면서 진행 중임.
	(커니) 12개 과제는 모든 계열사에서 쓸 수 있는 내용인 것인지?
	(배혜련 팀장) 맞음. 데이터 자체는 모든 계열사를 다룰 수 없어서 데이터가 제일 많은 회사의 케이스로 PoC를 돌리고 있음. 다 돌아가면 다른 회사도 사용할 수 있을 것 같음.
	(조욱희 수석) 데이터를 AI 학습에 쓸 수 있는 형태로 만드는 작업을 현재 CHRO에서는 못함. 예를 들어 현재 S3를 사용해서 칼럼명을 영문으로 바꿔야 하는데, 시스템에 원래 있는 영문명을 확보하지 못하고 한글을 자체적으로 영어로 번역해서 사용함. 그래서 나중에 실제 데이터와 연결할 때 문제가 있을 것 같음.
	(유지인 수석) 모듈원이 실제로 과제에서 활용하기 위해서는 테이블, 필드, 값이 어떤 의미인지 어려운 상황임. 
	(조욱희 수석) 과제 진행의 전제는 시스템에 뽑힌 데이터는 담당 GR이 알 수 있다고 생각해 굳이 설명을 붙이지 않았음. 변환된 영문명이 어떤 데이터의 제목이고 원본 데이터의 한글 칼럼이 뭐였는지 만들어서 주긴 했지만, 모듈원 입장에서는 미리 숙지가 됐으면 좋았을 텐데 중간중간 숙지가 되면서 진행 중임.
	(커니) PwC는 개인 정보 마스킹 관련된 것만 하는 것인지? 혹은 모든 데이터 전처리를 하는 것인지?
	(조욱희 수석) 마스킹해서 올리는 것만 한다고 보면 됨. 이메일 데이터 전처리 같은 작업과 그 데이터를 가지고 뭔가를 만드는 것은 60명의 현업이 함. CHROI는 데이터 수집과 가명 처리 가이드 같은 것을 줌.
	(두산지주) 마스킹 처리를 하면 모든 과제에 동일한 가명으로 표시되는지?
	(조욱희 수석) 키 값을 사번으로 통일함. 조직명이 연결돼야 하는 다른 데이터가 있으면 그것도 맞춰달라고 요청을 받아서 키 값이 사번이기 때문에 같은 코드를 돌리면 동일하게 진행된다고 함.
	(커니) 다시 복호화를 할 수 있는 것인지?
	(조욱희 수석) 그것까지는 모르겠음. 아직 복호화를 해서 볼 필요는 없음.
	(배혜련 팀장) 이런 프로젝트를 진행하기 전에 표준화된 프로세스와 체크리스트가 있어서 미리 확인하고 진행할 수 있도록 했으면 좋겠음. 
□	마무리
