AI/Data 과제에 필요한 데이터를 정의하거나 식별하는 과정은 원활했나요?	현업이 알고 있는 데이터와 시스템 데이터 간의 차이가 발생하는 경우가 있음	데이터 카탈로그	조직 내 데이터 자산 목록화를 통해 데이터 출처/소유자와 현업이 알고 있는 데이터 간의 협의/이해를 돕는다.	현업 데이터와 시스템 데이터 중 어떤 데이터를 자산화 및 목록화할 것인가?
AI/Data 과제 수행에 필요한 데이터는 어떤 방식으로 확보하셨나요? 이 과정에서 주요 병목은 무엇이었나요?	ERP, MES, 엑셀, 파일서버 등 여러 곳에서 데이터를 취합함	데이터 전처리	여러 형태로 존재하는 데이터를 정제/변환/정규화 과정을 통해 수월한 취합을 돕는다.	ERP, MES, 엑셀, 파일 서버 등 여러 형태로 존재하는 데이터를 어떻게 공식 데이터 자산으로 전환하고 지속하여 수집할 것인가?
필요한 데이터를 찾고, 이해하고, 접근하기 위한 관리 체계가 충분한지 의견을 작성해 주세요.	사람에 따라 다르지만 기간계 시스템의 경우 각 담당자가 있으므로 어렵지 않으나 비정형 데이터의 경우 접근 방식이 모두 다르므로 어려움	데이터 카탈로그	조직 내 데이터 자산 목록화를 통해 데이터의 접근 방식/경로를 명확하게 제공한다.	비정형 데이터의 각기 다른 접근 방식을 어떻게 검색/탐색 가능하게 만들 것인가?
지원하신 계열사에서 데이터 관련 의사결정(정의, 품질, 접근 권한 등)은 누가 담당하고 있었나요? 전담 조직·담당자가 있었던 경우와 없었던 경우 각각 어떤 차이가 있었는지 경험을 공유해 주세요.	-	-	-	-
AI/Data 과제에서 데이터의 신뢰도나 활용 가능성을 낮추는 품질 이슈는 무엇이었나요?	정형 데이터의 경우는 품질 이슈가 제한적이며, 비정형 데이터의 경우 추출부터 어떤 항목을 어떻게 추출할 것인가부터 필요없는 정보들은 어떻게 처리할 것인가 등 품질 이슈가 발생할 수 있음	데이터 전처리	비정형 데이터의 추출부터 필요 없는 정보의 처리에서 품질 이슈가 발생하므로 비정형 데이터의 정제/변환/정규화를 통해 해결한다.	비정형 데이터의 정제/변환/정규화를 위해 어떤 통일된 기준을 바탕으로 데이터 추출을 진행할 것인가?
보유한 데이터를 AI 과제에 적용하기 위해 추가로 보완이 필요한 부분이 무엇이었나요?	정형 데이터는 관련 데이터에 대한 지식을 요구하며, 비정형 데이터의 경우 데이터를 받더라도 어떻게 사용할지 정의가 어려움	데이터 해설/주석	사람이 이해할 수 있는 설명 등의 부여를 통해 정형 데이터의 도메인 지식 장벽을 해소하고 비정형 데이터의 사용 정의를 명확화하여 해결할 수 있다.	정형 데이터의 도메인 지식 장벽 해소와 비정형 데이터의 사용 정의 명확화를 위해 어느 정도 수준의 해설/주석이 필요한가?
수집된 데이터를 AI 학습데이터로 바로 활용하기 어려웠다면 그 이유와 사례를 작성해주세요	비정형 데이터 - 어떤 데이터를 추출할 것인가를 정의하기가 어려움	메타데이터	데이터의 구조, 형식, 추출 기준 등이 달라도 동일한 스키마를 준수할 수 있도록 관리한다.	데이터의 구조, 형식, 추출 기준 등을 관리하는 스키마를 어떻게 정의하고 관리할 것인가?
향후 AI/Data 활용을 확대하기 위해 그룹 차원에서 우선적으로 지원이 필요한 영역에 대해 자유롭게 작성해 주세요	솔루션 표준화 - 각 계열사별로 다르다보니 경험 이슈등에 대한 공유가 어려움	데이터 product화	계열사마다 사용하는 도구가 다르더라도 반복 활용 가능한 상품 형태로 데이터를 제공하여 동일한 언어/형식으로 경험/이슈를 공유할 수 있다.	계열사마다 다른 솔루션을 사용하는 상황에서 데이터를 어떻게 반복 활용 가능한 형태로 가공하고 제품 형태로 제공할 것인가?
