- 데이터의 용량이 커서 병합등 handling 하는데 시간이 많이 소요됨"
AI/Data 과제 수행에 필요한 데이터는 어떤 방식으로 확보하셨나요? 이 과정에서 주요 병목은 무엇이었나요?	"- ERP, MES, 엑셀, 파일서버 등 여러 곳에서 데이터를 취합함

- 권한 승인 절차로 인해 데이터 확보까지 시간이 오래 걸림
- 일부 데이터는 시스템이 아닌 수기 파일로 관리되고 있었음
- 데이터 추출 방식이 자동화되어 있지 않아 반복 수작업이 발생함"	"- Test 설비에서 DAQ 시스템을 통해 data가 local PC에 자동으로 저장되는 구조
- Raw 데이터는 초단위로 30분 정도 분량으로 하나의 파일이 생성되며, 파일별로 원하는 목적에 맞게 저장되지 않음으로 분류작업이 필요함
- 1차 가공된 데이터는 엑셀 Macro 형태로, Pass/Fail로 활용되며 수기로 관리되는 데이터가 있음, 설비의 트러블 등 Issue 상황은 별도 파일로 관리되고 있음
- 성능 데이터는 1차 가공되어 엑셀파일에 저장되는데 데이터 추출을 위해 별도 SW를 개발하여 데이터 추출 진행
- 제품 모델 종류에 따라 data의 이름이 달라 모델 별 별도 관리가 되고 있음 "
필요한 데이터를 찾고, 이해하고, 접근하기 위한 관리 체계가 충분한지 의견을 작성해 주세요.	"- 데이터 정의가 명확하지 않음
- 동일한 용어가 부서마다 다르게 사용됨
- 데이터 담당자 또는 오너가 불명확함
- 최신 데이터인지 확인하기 어려움
- 데이터 변경 이력이나 출처를 알기 어려움
- 데이터 품질을 정기적으로 점검하지 않음"	"- 설비에서 나오는 데이터가 database화 되어 검색조건에 따라 검색 및 다운로드 받으면 좋은데, 필요 할 때마다 현장에 가서 백업을 받아야 함
- 일부 공정 중 발생하는 데이터는 MES 시스템으로 저장이 되나, 변경 작업에 대한 유연성이 떨어짐"
"AI/Data 과제에서 데이터의 신뢰도나 활용 가능성을 낮추는 품질 이슈는 무엇이었나요?
(AI과제에서 “이 데이터는 믿고 쓰기 어렵다”고 판단된 경우 등)"	- 누락, 중복, 오류, 단위 불일치, 코드 불일치, 날짜/시점 불일치	- 과거에는 DAQ 시스템의 일시적 오류로 인한 Data 누락 발생이 있음
보유한 데이터를 AI 과제에 적용하기 위해 추가로 보완이 필요한 부분이 무엇이었나요?	"- 원하는 데이터를 어디서 찾아야 할지 모름
