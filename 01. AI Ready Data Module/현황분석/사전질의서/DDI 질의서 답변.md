AI/Data 과제를 수행하면서 가장 어려웠던 점은 무엇이었나요?	현업의 눈높이 조정
AI/Data 과제에 필요한 데이터를 정의하거나 식별하는 과정은 원활했나요?	현업이 알고 있는 데이터와 시스템 데이터 간의 차이가 발생하는 경우가 있음
AI/Data 과제 수행에 필요한 데이터는 어떤 방식으로 확보하셨나요? 이 과정에서 주요 병목은 무엇이었나요?	ERP, MES, 엑셀, 파일서버 등 여러 곳에서 데이터를 취합함
필요한 데이터를 찾고, 이해하고, 접근하기 위한 관리 체계가 충분한지 의견을 작성해 주세요.	사람에 따라 다르지만 기간계 시스템의 경우 각 담당자가 있으므로 어렵지 않으나 비정형 데이터의 경우 접근 방식이 모두 다르므로 어려움
지원하신 계열사에서 데이터 관련 의사결정(정의, 품질, 접근 권한 등)은 누가 담당하고 있었나요? 전담 조직·담당자가 있었던 경우와 없었던 경우 각각 어떤 차이가 있었는지 경험을 공유해 주세요.	-
"AI/Data 과제에서 데이터의 신뢰도나 활용 가능성을 낮추는 품질 이슈는 무엇이었나요?
(AI과제에서 “이 데이터는 믿고 쓰기 어렵다”고 판단된 경우 등)"	정형 데이터의 경우는 품질 이슈가 제한적이며, 비정형 데이터의 경우 추출부터 어떤 항목을 어떻게 추출할 것인가부터 필요없는 정보들은 어떻게 처리할 것인가등 품질 이슈가 발생할 수 있음
보유한 데이터를 AI 과제에 적용하기 위해 추가로 보완이 필요한 부분이 무엇이었나요?	정형 데이터는 관련 데이터에 대한 지식을 요구하며, 비정형 데이터의 경우 데이터를 받더라도 어떻게 사용할지 정의가 어려움
수집된 데이터를 AI 학습데이터로 바로 활용하기 어려웠다면 그 이유와 사례를 작성해주세요	비정형 데이터 - 어떤 데이터를 추출할 것인가를 정의하기가 어려움
향후 AI/Data 활용을 확대하기 위해 그룹 차원에서 우선적으로 지원이 필요한 영역에 대해 자유롭게 작성해 주세요	솔루션 표준화 - 각 계열사별로 다르다보니 경험 이슈등에 대한 공유가 어려움
