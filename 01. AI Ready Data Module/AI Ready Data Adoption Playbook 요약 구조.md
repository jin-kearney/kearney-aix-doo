## 전체 스토리라인

AI Ready Data Adoption Playbook은 계열사의 AI 활용 확대에 필요한 데이터 준비 체계를 정의하고, 이를 실행·운영 조직과 거버넌스로 연결하는 실무형 가이드다.

| 구분                                | 역할        | 핵심 메시지                                                               |
| --------------------------------- | --------- | -------------------------------------------------------------------- |
| **1. AI Ready Data 전체 Framework** | 전체 뚜껑     | 왜 20개 주제가 필요한지, 그리고 왜 이를 Drive할 조직과 R&R이 필요한지 설명한다.                  |
| **2. 20개 주제별 Topic Canvas**       | 주제별 실행 설계 | 20개 주제 각각에 대해 Why, What, When, Where, Who, How, KPI, Roadmap으로 정리한다. |
| **3. 계열사 실행·운영 Governance**       | 운영 체계     | 계열사 조직별 R&R과 DE팀 중심의 표준·진단·점검·KPI 관리 체계를 정의한다.                       |

# 1. AI Ready Data 전체 Framework

## 1.1 목적

1장은 백서의 전체 뚜껑이다.  
현재 AI 활용 현황을 보면 AI Ready Data 20개 주제가 모두 필요하며, 동시에 이를 지속적으로 관리할 조직과 R&R도 필요하다는 점을 설명한다.

---

## 1.2 현재 현황과 문제의식

계열사에서는 RAG, Agent, 예측 AI, 업무 자동화, 품질 분석 등 AI Use Case가 확대되고 있다.  
하지만 AI가 활용할 데이터는 아직 충분히 준비되어 있지 않다.

주요 문제는 다음과 같다.

|문제 영역|내용|
|---|---|
|데이터 탐색 어려움|필요한 데이터가 어디에 있는지 찾기 어려움|
|의미 불일치|같은 용어와 지표도 계열사·부서별로 다르게 해석됨|
|품질 불확실|데이터의 정확성, 최신성, 완전성을 신뢰하기 어려움|
|권한·보안 기준 미흡|AI가 어떤 데이터에 접근해도 되는지 기준이 불명확함|
|Lineage 부족|데이터가 어디서 왔고 어떻게 변환되었는지 추적하기 어려움|
|평가 데이터 부족|AI 결과를 검증할 기준 데이터와 테스트셋이 부족함|
|운영 주체 불명확|AI Ready Data를 누가 만들고 관리할지 R&R이 명확하지 않음|

---

## 1.3 20개 주제를 도출한 방식

AI Ready Data 20개 주제는 단순히 기능 목록을 나열한 것이 아니라, **Top-down 관점과 Bottom-up 관점**을 함께 반영해 도출한다.

|접근 방식|설명|목적|
|---|---|---|
|**Top-down ① Data Layer 관점**|데이터 수집, 저장, 가공, 관리, 활용, 운영 등 데이터 생애주기 각 단계에서 AI 활용을 위해 필요한 주제를 도출|데이터 흐름 전반에서 빠지는 영역이 없도록 함|
|**Top-down ② AI Era 관점**|기존 분석 AI뿐 아니라 RAG, Agent, 생성형 AI, Physical AI 등 AI 시대 변화에 따라 새롭게 필요한 데이터 준비 주제를 도출|AI 기술 변화에 맞는 신규 데이터 역량 반영|
|**Bottom-up 계열사 Pain Point 관점**|실제 계열사들이 겪는 데이터 문제를 수집하고, 이를 공통 유형으로 묶은 뒤 20개 주제에 매핑|현장 문제와 주제 체계의 실효성 검증|

즉, 20개 주제는 **이론적으로 필요한 데이터 역량**과 **현장에서 실제로 반복되는 문제**를 함께 반영한 체계다.

---

## 1.4 Bottom-up 문제 유형화

계열사별 문제를 개별 사례로만 나열하면 복잡해지므로, 백서에서는 이를 상위 Pain Point 유형으로 정리한다.

|계열사 데이터 문제 유형|설명|연결되는 AI Ready Data 주제 예시|
|---|---|---|
|**1. 데이터 발견·접근 문제**|필요한 데이터가 어디 있는지 모르고, 소유자와 접근 방법도 불명확함|**A1 카탈로그, A2 메타데이터, C2 품질/권한, F4 보안/마스킹**|
|**2. 데이터 의미·표준 문제**|용어, 지표, 코드, 업무 정의가 부서·시스템별로 다름|**A3 Glossary, B3 온톨로지, A2 메타데이터**|
|**3. 데이터 품질·신뢰 문제**|누락, 중복, 오류, 최신성 부족으로 AI 결과를 신뢰하기 어려움|**B1 전처리, C1 Observability, C2 품질/권한, C3 Lineage**|
|**4. 데이터 연결·맥락 부족 문제**|데이터 간 관계, 업무 맥락, 원천·변환 과정이 연결되어 있지 않음|**B2 주석, B3 온톨로지, C3 Lineage, A3 Glossary**|
|**5. 비정형·현장 데이터 활용 문제**|문서, 이미지, 센서, 설비, 현장 데이터가 AI가 쓰기 어려운 형태로 존재함|**B1 전처리, F1 Physical 데이터, A2 메타데이터, C1 Observability**|
|**6. AI 평가·검증 문제**|AI 결과가 맞는지 판단할 기준 데이터, 평가셋, 테스트 체계가 부족함|**E3 평가 데이터, F3 Prompt/Harness, E4 Feedback Loop**|
|**7. Agent·자동화 연계 문제**|AI가 실제 업무 시스템, API, Tool과 안전하게 연결되기 어려움|**F2 API/Tool 데이터, F3 Prompt/Harness, C2 품질/권한, F4 보안/마스킹**|
|**8. 데이터 운영·생명주기 문제**|데이터가 생성된 이후 변경, 폐기, 갱신, 품질 점검이 체계적으로 관리되지 않음|**D1 운영관리, D2 생명주기 관리, C1 Observability, E4 Feedback Loop**|
|**9. 조직·R&R 부재 문제**|데이터 오너, 현업, IT, 보안, AI 조직 간 책임과 의사결정 구조가 불명확함|**D1 운영관리, C2 품질/권한, D2 생명주기 관리 + 3장 Governance**|

이렇게 정리하면 Bottom-up은 단순 사례 수집이 아니라,  
**계열사 공통 문제 → AI Ready Data 주제 → 실행 Governance**로 연결된다.

---

## 1.5 AI 유형별 우선 주제

20개 주제는 모두 AI Ready Data 체계의 일부지만, AI 유형별로 우선순위는 다르다.  
따라서 계열사는 자사의 AI 추진 방향에 따라 먼저 집중해야 할 주제를 판단할 수 있어야 한다.

|AI 유형 / 추진 방향|우선적으로 중요한 AI Ready Data 주제|
|---|---|
|**문서 기반 RAG / 지식 검색**|**A1 카탈로그, A2 메타데이터, A3 Glossary, B1 전처리, B3 온톨로지, C3 Lineage, F4 보안/마스킹**|
|**업무 자동화 Agent**|**F2 API/Tool 데이터, F3 Prompt/Harness, C2 품질/권한, C3 Lineage, E3 평가 데이터, E4 Feedback Loop, F4 보안/마스킹**|
|**품질 원인 분석 Agent**|**A1 카탈로그, A3 Glossary, B2 주석, B3 온톨로지, C2 품질/권한, C3 Lineage, E3 평가 데이터, F3 Prompt/Harness**|
|**현장·설비 예측 AI**|**F1 Physical 데이터, C1 Observability, C2 품질/권한, D1 운영관리, E4 Feedback Loop**|
|**AI 학습 데이터 구축**|**B1 전처리, B2 해설/주석, C2 품질/권한, E2 합성데이터, E3 평가 데이터, F4 보안/마스킹**|
|**전사 데이터 기반 구축**|**A1 카탈로그, A2 메타데이터, A3 Glossary, C2 품질/권한, D1 운영관리, D2 생명주기 관리**|

핵심 메시지는 다음과 같다.

> 20개 주제는 전사 AI Ready Data 체계를 구성하는 전체 역량이다.  
> 다만 RAG, Agent, 예측 AI, 학습 데이터 구축 등 AI 유형에 따라 먼저 준비해야 할 주제와 적용 깊이는 달라진다.

# 2. 20개 주제별 Topic Canvas

## 목적

2장은 A1~F4의 20개 주제를 각각 동일한 Canvas 구조로 설명하는 장이다.  
각 주제를 단순 개념이 아니라 실행 가능한 준비 항목으로 정리한다.

## Canvas 구성

|항목|핵심 질문|담아야 할 내용|
|---|---|---|
|**Why**|왜 필요한가?|AI Ready Data 관점에서 필요한 이유, 없을 때 발생하는 문제|
|**What**|무엇인가?|주제의 정의, 범위, 고유 책임, 다른 주제와의 차이|
|**When**|언제 적용하는가?|필요한 Use Case, 적용 우선순위, 적용하지 않아도 되는 경우|
|**Where**|어디에 적용하는가?|데이터 수집·저장·가공·활용·운영 단계 중 적용 위치, 관련 시스템·업무 영역|
|**Who**|누가 책임지는가?|지주, 계열사, 데이터 오너, 현업, IT, 보안, AI 조직, DE팀의 주요 역할|
|**How**|어떻게 구현하는가?|준비 절차, 최소 요건, 표준, 운영 프로세스, 기술 방식, Tech Stack, 필요한 산출물|
|**KPI**|어떻게 관리하는가?|품질 지표, 활용 지표, 운영 지표, 성숙도 판단 기준|
|**Roadmap**|어떻게 확산하는가?|PoC, Pilot, 계열사 확산, 전사 표준화 단계|

## 작성 흐름

> **Why → What → When → Where → Who → How → KPI → Roadmap**

이 흐름을 통해 각 주제는  
**필요성 이해 → 개념 정의 → 적용 판단 → 실행 설계 → 운영 관리 → 확산 계획**으로 연결된다.

# 3. 계열사 실행·운영 Governance

## 목적

3장은 20개 주제를 실제 계열사 조직 안에서 누가 만들고, 누가 운영하며, DE팀이 어떻게 Drive할지 정의하는 장이다.

## 계열사 내 주요 역할

|조직 / 역할|주요 책임|
|---|---|
|**현업**|AI Use Case 정의, 업무 의미 제공, 결과 검증|
|**데이터 오너**|데이터 정의, 품질 책임, 접근 권한 승인|
|**IT / 데이터 플랫폼팀**|데이터 수집·저장·연계, 시스템 구현, 자동화|
|**보안 / 개인정보 조직**|민감정보 식별, 마스킹, 접근통제, 규제 검토|
|**AI / 분석 조직**|모델·Agent 개발, 평가 데이터 정의, 성능 검증|
|**계열사 데이터 조직**|계열사 내 표준, 품질, 카탈로그, Lineage 운영|
|**DE팀**|전사 표준 수립, 진단, 점검, KPI 관리, 우수사례 확산|

## DE팀의 역할

|DE팀 역할|내용|
|---|---|
|**표준 수립**|20개 주제별 정의, 최소 요건, 템플릿, KPI 기준 제공|
|**진단**|계열사별 AI Ready Data 성숙도와 부족 역량 진단|
|**우선순위 지원**|AI 유형과 계열사 상황에 맞는 추진 순서 제안|
|**이행 점검**|주제별 준비 수준, 품질 기준, 운영 프로세스 점검|
|**KPI 관리**|품질, 활용, 보안, 운영 지표 관리|
|**거버넌스 운영**|정기 리뷰, 변경관리, 예외 승인, 이슈 조정|
|**확산 지원**|우수사례, 공통 도구, 교육, 표준 템플릿 전파|

---

# 최종 요약 문장

> **AI Ready Data Adoption Playbook은 Top-down으로 데이터 생애주기와 AI 시대 변화에서 필요한 주제를 도출하고, Bottom-up으로 계열사의 데이터 Pain Point를 유형화해 20개 주제와 매핑한 뒤, 이를 주제별 Canvas와 DE팀 중심 Governance로 실행·운영하는 가이드다.**
