### Bottom-up 문제 유형화

| 계열사 데이터 문제 유형           | 설명                                            | 연결되는 AI Ready Data 주제 예시                                    |
| ----------------------- | --------------------------------------------- | ----------------------------------------------------------- |
| **1. 데이터 발견·접근 문제**     | 필요한 데이터가 어디 있는지 모르고, 소유자와 접근 방법도 불명확함         | **A1 카탈로그, A2 메타데이터, C2 품질/권한, F4 보안/마스킹**                  |
| **2. 데이터 의미·표준 문제**     | 용어, 지표, 코드, 업무 정의가 부서·시스템별로 다름                | **A3 Glossary, B3 온톨로지, A2 메타데이터**                          |
| **3. 데이터 품질·신뢰 문제**     | 누락, 중복, 오류, 최신성 부족으로 AI 결과를 신뢰하기 어려움          | **B1 전처리, C1 Observability, C2 품질/권한, C3 Lineage**          |
| **4. 데이터 연결·맥락 부족 문제**  | 데이터 간 관계, 업무 맥락, 원천·변환 과정이 연결되어 있지 않음         | **B2 주석, B3 온톨로지, C3 Lineage, A3 Glossary**                 |
| **5. 비정형·현장 데이터 활용 문제** | 문서, 이미지, 센서, 설비, 현장 데이터가 AI가 쓰기 어려운 형태로 존재함   | **B1 전처리, F1 Physical 데이터, A2 메타데이터, C1 Observability**     |
| **6. AI 평가·검증 문제**      | AI 결과가 맞는지 판단할 기준 데이터, 평가셋, 테스트 체계가 부족함       | **E3 평가 데이터, F3 Prompt/Harness, E4 Feedback Loop**          |
| **7. Agent·자동화 연계 문제**  | AI가 실제 업무 시스템, API, Tool과 안전하게 연결되기 어려움       | **F2 API/Tool 데이터, F3 Prompt/Harness, C2 품질/권한, F4 보안/마스킹** |
| **8. 데이터 운영·생명주기 문제**   | 데이터가 생성된 이후 변경, 폐기, 갱신, 품질 점검이 체계적으로 관리되지 않음  | **D1 운영관리, D2 생명주기 관리, C1 Observability, E4 Feedback Loop** |
| **9. 조직·R&R 부재 문제**     | 데이터 오너, 현업, IT, 보안, AI 조직 간 책임과 의사결정 구조가 불명확함 | **D1 운영관리, C2 품질/권한, D2 생명주기 관리 + 3장 Governance**           |
## 구조적 문제점 정의서

Bottom-Up 

| 문제 영역        | 내용                                                  |
| ------------ | --------------------------------------------------- |
| 1. 데이터 준비 부족 | 데이터 위치, 의미, 품질, 권한, Lineage, 평가 데이터가 충분히 정비되어 있지 않음 |
| 2. 조직·R&R 부재 | AI Ready Data를 누가 만들고, 승인하고, 운영하고, 개선할지 명확하지 않음     |
| 3. 운영 체계 미흡  | AI 데이터 준비가 과제별·수작업 중심으로 이루어지고, 지속 관리 체계가 부족함        |

### 각 주제별 Canvas/Storyline 정의

| 항목          | 핵심 질문      | 담아야 할 내용                                              |
| ----------- | ---------- | ----------------------------------------------------- |
| **Why**     | 왜 필요한가?    | AI Ready Data 관점에서 필요한 이유, 없을 때 발생하는 문제               |
| **What**    | 무엇인가?      | 주제의 정의, 범위, 고유 책임, 다른 주제와의 차이                         |
| **When**    | 언제 적용하는가?  | 필요한 Use Case, 적용 우선순위, 적용하지 않아도 되는 경우                 |
| **Where**   | 어디에 적용하는가? | 데이터 수집·저장·가공·활용·운영 단계 중 적용 위치, 관련 시스템·업무 영역           |
| **Who**     | 누가 책임지는가?  | 지주, 계열사, 데이터 오너, 현업, IT, 보안, AI 조직, DE팀의 주요 역할        |
| **How**     | 어떻게 구현하는가? | 준비 절차, 최소 요건, 표준, 운영 프로세스, 기술 방식, Tech Stack, 필요한 산출물 |
| **KPI**     | 어떻게 관리하는가? | 품질 지표, 활용 지표, 운영 지표, 성숙도 판단 기준                        |
| **Roadmap** | 어떻게 확산하는가? | PoC, Pilot, 계열사 확산, 전사 표준화 단계                         |
