# 최종 산출물 스토리라인

## AI Ready Data Adoption Playbook

백서는 크게 3단 구조로 가는 것이 좋습니다.

| Layer              | 목적                      | 백서에 담을 내용                                   |
| ------------------ | ----------------------- | ------------------------------------------- |
| 1. 판단 Framework    | 우리 계열사에 무엇이 필요한가?       | Use Case 유형별로 필요한 AI Ready Data 주제 선택       |
| 2. 주제별 Canvas      | 각 주제를 어떻게 준비해야 하는가?     | What / Why / When / Who / How / Where 중심 정리 |
| 3. 실행·운영 Framework | 실제로 누가 만들고 어떻게 운영할 것인가? | RACI, 산출물, KPI, 성숙도, Roadmap                |

즉, 백서는 **A1~F4를 전부 동일 깊이로 설명하는 방식**이 아니라,  
계열사가 먼저 **자기 AI Use Case를 보고 필요한 주제를 고른 뒤**, 선택된 주제에 대해 **실행 Canvas**를 따라 체계를 만드는 구조가 좋습니다.

---

## 1. 판단 Framework

### “우리에게 어떤 AI Ready Data 주제가 필요한가?”

계열사가 가장 먼저 해야 할 일은 “20개 주제를 모두 해야 하나?”가 아니라,  
**우리 AI 과제에 필요한 데이터 역량이 무엇인지 선택하는 것**입니다.

그래서 백서 앞부분에 아래 같은 **Use Case 기반 적용 Matrix**가 필요합니다.

| AI Use Case 유형    | 필요한 AI Ready Data 주제                                                                          |
| ----------------- | --------------------------------------------------------------------------------------------- |
| 문서 기반 RAG / 지식 검색 | A1 카탈로그, A2 메타데이터, A3 Glossary, B1 전처리, B3 온톨로지, C3 Lineage, F4 보안/마스킹                        |
| 품질 원인 분석 Agent    | A1 카탈로그, A3 Glossary, B2 주석, B3 온톨로지, C2 품질/권한, C3 Lineage, E3 평가 데이터, F3 Prompt/Harness      |
| 현장·설비 예측 AI       | F1 Physical 데이터, C1 Observability, C2 품질/권한, D1 운영관리, E4 Feedback Loop                        |
| 업무 자동화 Agent      | F2 API/Tool 데이터, F3 Prompt/Harness, C2 권한, C3 Lineage, E3 평가 데이터, E4 Feedback Loop, F4 보안/마스킹 |
| AI 학습 데이터 구축      | B1 전처리, B2 해설/주석, C2 품질/권한, E2 합성데이터, E3 평가 데이터, F4 보안/마스킹                                    |
| 전사 데이터 기반 구축      | A1 카탈로그, A2 메타데이터, A3 Glossary, C2 품질/권한, D1 운영관리, D2 생명주기 관리                                 |

이 Matrix가 있어야 계열사 입장에서  
“우리 과제는 RAG니까 모든 걸 다 할 필요는 없고, 우선 A1, A2, B1, C3, F4부터 해야겠구나”라고 판단할 수 있습니다.

## 2. 주제별 Canvas

### What / Why / When / Who / How / Where 구조

각 주제는 길게 설명하기보다, 아래 Canvas 형태로 정리하는 것이 좋습니다.

## AI Ready Data Topic Canvas

| 항목     | 백서에 담을 내용                                         |
| ------ | ------------------------------------------------- |
| What   | 이 주제가 무엇인지, 고유 책임이 무엇인지                           |
| Why    | AI Ready Data 관점에서 왜 필요한지, 없으면 어떤 문제가 생기는지        |
| When   | 어떤 경우에 이 주제를 적용해야 하는지, 적용하지 않아도 되는 경우는 무엇인지       |
| Who    | 지주, 계열사, 데이터 오너, 현업, IT, 보안, AI 조직이 각각 무엇을 해야 하는지 |
| How    | 준비 절차, 최소 요건, 표준, 기술 방식, 운영 프로세스                  |
| Where  | 데이터 수집·관리·활용 단계 중 어디에 적용되는지, 어떤 시스템·업무 영역에 위치하는지  |
| Output | 실제 만들어야 하는 산출물                                    |
| KPI    | 관리 지표와 성숙도 판단 기준                                  |
