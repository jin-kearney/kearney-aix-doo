AI Ready Data Module

AI 과제 유형화와 데이터 준비 프레임워크

AI Ready Data Module의 문서를 기반으로, 실행 관점의 발표 흐름으로 재구성한 페이지입니다.

1. AI 과제 2×2 유형화

기준 축: Insight AI(판단·예측·분류·추천) × Action AI(생성·처리·실행·제어), Business/Digital × Physical/Industrial

업무 인사이트형 (Insight Strategist)

고객·문서·영업·서비스·경영 데이터를 분석해 판단/추천을 지원

업무 실행형 (Enterprise Copilot)

문서 작성, 고객 응대, 업무 처리, 프로세스 자동화 수행

현장 인사이트형 (Field Analyst)

설비·공정·품질·현장 데이터를 기반으로 예측·진단

현장 실행형 (Autonomous Controller)

설비·로봇·장비·공정을 제어하고 작업 자동 실행

2. 유형별 데이터/기술 스택

업무 인사이트형

데이터: CRM, VOC, 계약서, KPI, ERP, 이메일/메신저
기술: NLP, 분류·예측, 추천, 이상탐지, BI 연계

업무 실행형

데이터: 문서/보고서/문의/매뉴얼/지식DB
기술: GenAI, LLM, RAG, RPA, 워크플로우 자동화

현장 인사이트형

데이터: 센서, 설비 로그, 영상, 품질 검사, 공정 조건
기술: 컴퓨터비전, 시계열, 예지보전, Edge AI

현장 실행형

데이터: MES/SCADA/PLC, 로봇 동작, 디지털트윈
기술: 폐루프 제어, 실시간 의사결정, OT/IT 통합, Fail-safe

3. 데이터 준비 5대 Framework (14개 주제)

Findable

  • 데이터 카탈로그
  • 메타데이터
  • 비즈니스 Glossary

Understandable

  • 데이터 전처리
  • 데이터 해설/주석
  • 온톨로지

Trustworthy

  • Observability
  • 데이터 품질/권한관리
  • 데이터 계통(Lineage)

Governed

  • 데이터 생명주기 관리
  • 데이터 운영관리
  • 데이터 Product화

Scaleable

  • 합성데이터 (Synthetic)
  • Physical 데이터

4. 발표 결론: 실행 우선순위

  1. Step 1 과제를 2×2로 분류하고 KPI를 과제별로 정의
  2. Step 2 5대 Framework를 기준으로 데이터 준비 상태(14개)를 진단
  3. Step 3 Trustworthy/Governed 항목부터 운영 통제 강화 후 확장
