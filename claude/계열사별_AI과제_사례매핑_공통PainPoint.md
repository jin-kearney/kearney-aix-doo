## Part 2. 비슷한 과제 유형에서 여러 계열사가 동일하게 겪은 어려움

### 유형 ① 문서/지식 LLM·RAG (챗봇·Agent) — 6개사 공통
**"문서는 많은데 AI가 읽을 수 있는 문서가 없다"**

| 공통 어려움                                                   | 겪은 계열사 (근거)                                                                                                                    |
| -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| 문서가 팀즈·개인폴더·이메일에 파편화, 포맷 비표준(PDF/Word/동영상/Excel 혼재)      | 테스나(매뉴얼 수집 2주 수작업), 에너빌리티(Teams 자료 위치 못 찾음), 전자BG(성적서·조치보고서 형식 상이), CHRO(PPT·이메일 개인 PC 관리), 밥캣(여러 버전 문서 혼재)                    |
| PDF 파싱·OCR·청킹의 표준 파이프라인이 없어 과제마다 처음부터 직접 설계              | 밥캣(청킹 커스텀 설계 6개월), 에너빌리티(구형 스캔 PDF 전처리 불가, 파서·OCR이 최장 소요 작업), 전자BG(범용 OCR 파이프라인 자체 구축), DDI(이물 PDF 파싱 로직 신규 개발), CHRO(컨설팅사 의존) |
| 이미지·도면을 AI가 해석 못함 (선 인식 오류, 비표준 포맷, 이미지-텍스트 관계 메타데이터 부재) | 에너빌리티(P&ID 딥러닝 추출 오류, DWG 파싱 불가), 밥캣(전장 도면 "완벽하게 이해할 수 있는 수준 아님", 매뉴얼 이미지 위치 혼동), 에너빌리티 데이터분석팀("도면을 이해하는데 정확도 한계")             |
| 비정형 데이터에서 "무엇을 추출해야 하는지" 정의 자체가 불가                       | DDI("원하는 데이터가 이메일 안에 있을지 없을지 모르기 때문에 추출하지 않음"), CHRO(이메일 비식별화 방법 모름), 에너빌리티("첨부 참조"만 기재된 레코드 결측 처리)                            |

### 유형 ② 시계열/설비데이터 분석·예지보전 — 4개사 공통
**"시스템이 실적 관리용으로 설계되어 AI용 공정 데이터가 원천적으로 없다"**

| 공통 어려움 | 겪은 계열사 (근거) |

### 유형 ③ 업무자동화 Agent·시스템 연계 — 5개사 공통
**"API가 없고, 권한이 없고, 요청하면 비용이 든다"**

| 공통 어려움 | 겪은 계열사 (근거) |

| 사내·자체 개발 시스템의 API 부재 → 자동화 시도가 좌초되거나 우회(RPA·CSV) | VCC(SAP "API 제공이 아예 없음" + DDI 개발 비용 → "차라리 하지 않는 쪽으로"), 에너빌리티(API 없어 "매번 다운하거나 RPA를 직접 만들어 사용"), 전자BG("외부인에게 API를 열어준다는 것 자체가 리스크"로 인식) |

## Part 3. 시사점 요약 (3줄)

1. **과제 유형이 달라도 막히는 지점은 같다** — 8개 조직 전부가 "데이터를 못 찾고(카탈로그), 못 연결하고(키/온톨로지), 못 읽히는(비정형 표준)" 동일한 3대 벽에서 좌초. 과제별 개별 해결이 아닌 그룹 공통 AI-Ready Data 체계가 필요한 근거.
2. **AI 기술이 아니라 데이터 준비에서 실패한다** — S&OP→PI 전환(전자BG/VCC), AVI 투자 홀딩(테스나), 트러블슈팅 PoC 중단(테스나), 이상탐지 학습 실패(퓨얼셀) 모두 모델이 아닌 데이터 단계에서 막힘. 전자BG는 과제 기간의 1/3이 데이터 확인, CHRO는 데이터 준비 기간 초과.
3. **같은 파이프라인을 계열사마다 다시 만들고 있다** — PDF 파싱/OCR/청킹을 밥캣(6개월)·에너빌리티·전자BG·DDI·CHRO가 각각 중복 개발. DDI 스스로 "솔루션 표준화 부재로 경험·이슈 공유가 어려움"이라 진단.
