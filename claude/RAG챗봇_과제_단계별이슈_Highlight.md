# RAG/Chatbot 구축 사례 — 과제 × 구축 단계 통합표

> 대상: RAG/Chatbot 구축과 직접 연결되는 사례 6건 (CHRO 비정형 PoC, DDI 재고 에이전트, 밥캣 Circuit Agent는 RAG 챗봇이 아니므로 제외)
> 원천: 인터뷰 회의록 (0521 에너빌리티, 0528 테스나, 0528 DDI, 0529 밥캣) + 사전질의서 통합본
>
> **에너빌리티 정정 반영**: 에너빌리티 인터뷰는 과제별이 아닌 주제별(카탈로그→전처리→온톨로지→품질)로 진행됨. 과제 단위로 이슈가 확인되는 것은 **GT 온톨로지(김용화 수석)뿐**이며, 나머지 2건은 발언자 기준으로 기재 — 이성민 수석(사내 RAG 다수 + 구형 PDF), 정성문 수석(외주문서 검증 불가·사진 미활용, 품질 업무 전반에 대한 발언으로 특정 과제에 귀속하지 않음. 이전 정리에서 QFR Agent에 묶었던 것을 정정).

---

## 1. RAG/Chatbot 구축의 일반 Step

```
S1 데이터 취합        문서가 어디 있는지 파악하고 한곳에 모음
S2 전처리·구조화      파싱/OCR → 청킹 → 메타데이터 부여
S3 지식화·인덱싱      임베딩, 온톨로지/시맨틱 레이어, retrieve 설계
S4 AI 개발            RAG 파이프라인·Agent 구축
S5 검증               정답셋 기반 답변 품질 평가
S6 운영·유지          문서 갱신·버전 관리, 품질 모니터링
```

## 3. 어려움이 어떻게 연결되는가

**Ⓑ "사람은 읽지만 AI는 못 읽는 문서" — 6건 중 5건의 S2에 등장 (최다)**
밥캣은 매뉴얼 PDF의 이미지-텍스트, 에너빌리티는 스캔 PDF와 도면의 선, 정성문 발언은 외주업체 제각각 양식, 전자BG는 이메일·실험데이터 양식. 문서 종류만 다를 뿐 "표준 없이 만들어진 문서를 AI에 넣기 전에 사람이 다시 정리한다"는 동일한 어려움이며, 각 사가 같은 파싱·전처리 작업을 처음부터 중복 수행.

**Ⓐ → Ⓒ → 중단의 연쇄 (테스나가 전형)**
자료 소재 파악(Ⓐ)에 2주를 쓰고 시작 → 지식 구조(Ⓒ) 없이 연결하니 답이 느리고 부정확 → 중단. 전자BG 이메일은 Ⓐ 단계에서 이미 "추출하지 않음"으로 범위가 축소된 상태로 진행 중.

**Ⓓ·Ⓔ는 '완주한' 과제에서만 드러남 (밥캣, 정성문 품질문서)**
앞 단계를 통과한 과제만 검증·운영 단계의 공백을 만남 — 정답셋이 없어 현업이 1,000개 질문을 만들거나(밥캣), 검증을 포기(정성문). 즉 S1~S3를 해결해도 Ⓓ·Ⓔ가 다음 병목으로 예고되어 있음.

**S4(AI 개발)는 전원 통과 ⭐**
6건 모두 RAG 파이프라인·Agent 구현 자체는 해냈음. 실패·지연·중단은 전부 개발의 앞(S1~S3)과 뒤(S5~S6)에서 발생.

---

## 4. 장표 메시지

6개 사례 모두 AI 개발(S4)이 아니라 **전처리(S2)·지식화(S3)에서 시간을 다 쓰거나 중단**됐고, 완주한 과제는 검증(S5)·운영(S6)의 공백을 추가로 만났다. S2·S3·S5·S6이 바로 AI-Ready Data 체계(표준 파싱 파이프라인·메타데이터·온톨로지·검증 데이터셋·버전 관리)가 그룹 공통으로 대신해줘야 할 영역 — 밥캣의 "최초 6개월 → 재진행 시 1~2달"이 그 효과의 방증.
