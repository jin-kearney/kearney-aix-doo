# E-1 데이터 Product화 (Data as a Product) — 스토리라인 요약 (콘텐츠 순서표)

> storyline.md를 펼치기 전 "무엇을, 어떤 순서로" 담는지 한눈에 보는 목차.
> 슬라이드 제목·핵심 메시지는 storyline.md와 일치한다.

## 한 줄 정의

데이터 Product화란, 반복 재사용 가능한 데이터를 오너·품질(SLA)·명세된 제공 인터페이스(Output Port)를 갖춘 '상품 단위'로 정의·운영해, 한 번 만들면 여러 팀과 AI가 신뢰하고 재사용하도록 하는 체계다.

## 전체 흐름

데이터를 '추출물'이 아닌 '제품'으로 보는 사고 전환과, 그 구조(오너십·Output Port·계약·SLA·검색가능성)가 없을 때 무엇이 막히는지로 필요성을 논증한다(Why) → 모든 데이터가 아니라 트리거가 보일 때 우선순위로 적용함을 짚고(When) → Data Product가 정확히 무엇이고 어떻게 구성·표준화되는지 개념을 완성한 뒤(What) → 후보 선정부터 명세·추상화·소비 포트·AI 자동화·솔루션까지 구축 방법을 제시하고(How) → 무엇으로 운영을 관리하며(KPI) → 어디까지 확산·고도화할지(Roadmap)로 마무리한다.

## 이 발표의 핵심 메시지 3가지

1. **데이터를 '제품'으로 보는 순간 책임(오너)·인터페이스(Output Port)·품질 약속(SLA)·검색가능성이 따라온다** — 이 구조가 없으면 재사용·품질·AI 신뢰가 원리적으로 불가능하다.
2. **모든 데이터를 제품화하지 않는다** — 반복 요청·다팀 사용·AI 활용·SLA 요구 같은 트리거가 보이는 데이터를, 카탈로그·메타데이터 기반 위에서 우선순위로 제품화한다.
3. **명세·계약·추상화·SLA 모니터링의 수작업은 LLM·이상탐지로 자동화하고**, 카탈로그→마켓플레이스→AI Agent 자율 소비(MCP)로 단계적으로 고도화한다.
